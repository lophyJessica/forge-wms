# Forge WMS 仓库管理系统

> 项目类型：B端供应链 | 角色：产品经理 | 周期：2021-2024
>
> 基于真实 WMS 产品经验，以Forge为演示品牌还原。

## 项目简介

Forge自研 WMS 仓库管理系统，支撑企业仓储作业全流程数字化。覆盖入库、出库、库存管理、盘点、调拨等核心模块，向上对接Forge 采购/销售系统，向下指导 PDA 手持终端现场作业。全国 6 个仓库，日出库 20,000+ 单。本人负责从需求调研到上线交付的全流程产品设计。

## 核心规范

1. 所有文档用简体中文输出，字段名、组件名、枚举值保留英文
2. PRD 文档必须包含：业务背景、功能范围、单据定位、业务场景、状态机、字段清单
3. 状态变更必须通过动作按钮触发
4. 主数据不提供删除，统一用"停用"操作替代
5. 业务流程梳理必须包含：业务流程图(Mermaid flowchart) + 系统时序图(Mermaid sequence)
6. 数据冲突时以简历为准，不编造经历数据

## 约束（NIS：Not In Scope）

- 本项目不涉及前端 UI 实现，仅负责产品设计层面
- 不涉及与第三方物流系统（快递/运输）的对接
- 不涉及硬件选型（PDA、扫码枪、打印机）
- 原型仅做前端静态页面，不写后端逻辑

## 技术栈（Demo 版）

- 前端：React + TypeScript + Tailwind CSS
- 存储：Dexie.js（IndexedDB 前端数据库，替代 localStorage）
- 构建：Vite
- 真实项目技术栈参考：React + Spring Boot + MySQL 8.0 + Android PDA
