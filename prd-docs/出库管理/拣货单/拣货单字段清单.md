# 拣货单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为拣货单字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- PICK 单号、来源波次、状态、应拣数量、商品快照、推荐货位不可手工编辑。
- 实拣数量、缺货数量按 `11-字段校验规范`：正整数或 0，确认时不得超过应拣上限。
- PDA 扫码字段优先来自扫描，不鼓励手工输入；异常补录需权限。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 空值展示 `-`，备注/原因说明 ≤200 字符。

| pickMode | 拣货模式 | enum | 继承 WAVE | 是 | 否 | 继承 WAVE 对应的 pickMode 字段枚举 |
| pickerId | 拣货员ID | string | 继承 WAVE/主管分配 | 是 | 否 | 必须为启用账号 |

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |

| lineNo | 行号 | integer | 系统生成 | 是 | 否 | 按推荐路径排序 |

| pickedAt | 明细完成时间 | datetime | 系统 | 否 | 否 | 明细确认时写入 |

## 5. 枚举值

### 5.1 拣货状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING_PICK | 待拣 | WAVE 已下推，等待 PDA 领取或开始 |
| PICKING | 拣货中 | PDA 已领取或已有明细确认 |
| PICKED | 已拣完 | 所有明细已拣或缺货确认完成 |
| VOIDED | 已作废 | 上游波次被关闭或异常终止，任务取消 |

### 5.2 明细状态 `lineStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING | 待拣 | 尚未开始该明细 |
| PICKING | 拣货中 | 已扫货位/商品但未确认完成 |
| PICKED | 已拣 | 实拣数量=应拣数量 |
| SHORTAGE | 缺货完成 | 实拣数量<应拣数量，且已登记缺货原因 |

### 5.3 缺货原因 `shortageReason`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| LOCATION_EMPTY | 货位无货 | 推荐货位无库存 |
| STOCK_LESS | 库存不足 | 货位库存少于应拣数量 |
| DAMAGED | 商品破损 | 现场发现破损不可拣 |
| BARCODE_UNREADABLE | 条码不可识别 | 商品或货位条码无法识别 |
| OTHER | 其他 | 需填写行备注 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 应拣总数 | `Σ shouldPickQty` | 拣货任务总量 |
| 实拣总数 | `Σ actualPickQty` | 已确认实拣数量合计 |
| 缺货数量 | `shouldPickQty - actualPickQty` | 明细级，不能为负数 |
| 缺货总数 | `Σ shortageQty` | 头部汇总 |
| 已处理数量 | `Σ actualPickQty + Σ shortageQty` | 用于判断是否全部处理 |
| 完成条件 | `已处理数量 = 应拣总数` | 所有明细实拣或缺货确认后完成 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| PDA 扫货位 | 货位条码是否匹配当前推荐货位 |
| PDA 扫商品 | 商品条码是否匹配当前 SKU |
| 数量确认 | 实拣数量为整数，且 `0≤实拣≤应拣` |
| 缺货提交 | 缺货数量>0 时缺货原因必填；原因=其他时行备注必填 |
| 完成拣货 | 所有明细状态为已拣或缺货完成 |
