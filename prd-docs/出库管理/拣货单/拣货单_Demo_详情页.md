# 拣货单_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看拣货单来源、明细、扫码记录、缺货异常和下游复核关联。

## 1. 页面定位

拣货单详情页面向研发、测试、产品复核，用于核对 PICK 是否由 WAVE 下推、扫码校验是否留下记录、实拣和缺货数据是否正确汇总，并确认库存过账没有在 PICK 阶段发生。

## 2. 页面结构

```text
拣货单详情
├─ 顶部摘要
│  ├─ 拣货单号、状态、异常标记
│  └─ 返回列表、查看来源波次、查看复核单
├─ 拣货头信息
│  ├─ 来源波次、仓库、拣货员、拣货模式
│  └─ 应拣总数、实拣总数、缺货总数、进度
├─ 拣货明细
│  ├─ 货位、商品、应拣数、实拣数、缺货数、明细状态
│  └─ 缺货原因、完成时间
├─ 扫码记录
│  ├─ 扫货位记录
│  ├─ 扫商品记录
│  └─ 错扫/阻断记录
├─ 关联进度
│  └─ 复核单号、复核状态
└─ 操作记录
   └─ 生成、领取、扫码、确认、缺货、完成
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态只展示 PICK 自身状态：待拣、拣货中、已拣完、已作废 |
| 来源信息 | 来源波次只读，可跳转 WAVE 详情 |
| 拣货明细 | 数量右对齐；缺货数量>0 的行标记异常 |
| 扫码记录 | 展示成功记录和阻断记录，便于排查错扫 |
| 关联进度 | 只读展示 CHECK，不在本页执行复核 |
| 库存说明 | 展示“PICK 阶段不扣减现存，包装完成扣减现存并释放占用” |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 查看来源波次 | 有 source_wave_no | 跳转 WAVE 详情 |
| 查看复核单 | 有 check_no | 跳转 CHECK 详情 |
| 返回列表 | 全部状态 | 返回拣货单列表 |

> 详情页不展示新增、审核、反审核、包装、交运操作。

## 6. 测试关注点

- PICK 详情能追溯来源 WAVE。
- 状态不出现待审核、已审核等审核流状态。
- 缺货行能展示缺货数量和原因。
- 错扫货位、错扫商品有操作记录。
- PICK 完成后只读，不允许继续修改实拣数量。
- PICK 阶段不生成 FL；库存扣减结果应在包装完成后的库存流水中查看。
