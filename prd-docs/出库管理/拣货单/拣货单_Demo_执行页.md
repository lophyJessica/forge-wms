# 拣货单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页为 PDA 拣货作业页，重点描述扫码交互；字段定义以 `拣货单字段清单.md` 为准。

## 1. 页面定位

PDA 拣货执行页用于拣货员领取由 WAVE 下推的 PICK，并按系统推荐路径完成“扫货位、扫商品、确认数量、登记缺货”。页面适合手套操作，使用大字体、大按钮，扫码优先于手动输入。

## 2. 页面结构

```text
PDA 拣货作业页
├─ 任务头
│  ├─ 拣货单号、来源波次、进度
│  └─ 仓库、拣货模式、拣货员
├─ 当前任务卡片
│  ├─ 推荐货位、商品名称、规格、单位
│  ├─ 应拣数量、已拣数量、缺货数量
│  └─ 路径序号：第 N / 共 M 个货位
├─ 扫码区
│  ├─ 扫货位
│  └─ 扫商品
├─ 数量区
│  ├─ 实拣数量输入
│  ├─ 加减步进
│  └─ 全拣快捷按钮
├─ 异常区
│  ├─ 缺货按钮
│  └─ 缺货原因选择
└─ 底部大按钮
   ├─ 确认本行
   └─ 完成拣货
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 领取任务 | 待拣任务进入拣货中，写入领取时间 |
| 扫货位 | 匹配当前推荐货位后才允许扫商品 |
| 扫商品 | 匹配当前 SKU 后才允许确认数量 |
| 全拣快捷按钮 | 将实拣数量填为应拣数量 |
| 数量步进 | 每次 +1/-1，不能小于 0，不能大于应拣数量 |
| 缺货 | 实拣少于应拣时选择原因并确认 |
| 完成拣货 | 全部明细已拣或缺货完成后展示 |

## 4. 扫码反馈

| 场景 | 反馈 |
|:--|:--|
| 扫货位正确 | 高亮货位，短提示音 |
| 扫错货位 | 红色提示、震动、语音提示“货位不匹配” |
| 扫商品正确 | 高亮商品，允许录数量 |
| 扫错商品 | 红色提示、震动、语音提示“商品不匹配” |
| 超拣 | 数量输入框标红，提示“实拣数量不能大于应拣数量” |
| 缺货未选原因 | 阻断提交，提示“请登记缺货原因” |

## 5. 页面状态

| PICK 状态 | 页面表现 |
|:--|:--|
| 待拣 | 展示领取任务按钮 |
| 拣货中 | 展示当前任务卡片和扫码区 |
| 已拣完 | 只读展示完成摘要，不允许继续扫码 |
| 已作废 | 只读展示“任务已作废”，显示“该任务已被作废或上游波次已关闭”，不允许扫码 |

## 7. 离线与同步

- PDA 支持离线缓存扫描记录，网络恢复后自动同步。
- 离线期间仍在本地执行基础校验：货位、商品、数量上限。
- 同步时以后端最新状态为准；若 PICK 已完成或明细已被处理，提示重新同步任务。
- 离线记录不得绕过超拣拦截。

## 8. 不展示内容

- 不展示新增拣货单入口。
- 不展示审核按钮。
- 不展示复核、包装、交运执行控件。
