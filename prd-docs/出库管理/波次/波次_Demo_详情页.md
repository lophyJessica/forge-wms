# 波次_Demo_详情页

> 角色：Demo·详情页 | 类型：业务单据
> 详情页用于查看波次头、波次明细、系统统计、关联下游进度和操作记录。

## 1. 页面定位

波次详情页服务研发、测试、产品复核，用于确认一个 WAVE 的聚合口径是否正确、明细是否完整、状态是否按按钮流转、下游进度是否只读展示。

## 2. 页面结构

```text
波次详情
├─ 顶部摘要
│  ├─ 波次单号、状态、类型
│  └─ 分配拣货员、下推拣货、关闭、返回列表
├─ 波次头信息
│  ├─ 仓库、承运商、线路、优先级、拣货模式、拣货员
│  └─ 订单数、SKU数、总件数、计划拣货日期、备注
├─ 波次明细
│  └─ 出库需求、SO行、SKU、订单数量、占用数量、波次数量
├─ 关联进度
│  ├─ PICK 拣货进度
│  ├─ CHECK 复核进度
│  ├─ PKG 包装进度
│  └─ DSH 交运进度
└─ 操作记录
   └─ 创建、分配、下推、关闭、完成回写
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态展示波次自身状态，不展示下游单据状态作为波次状态 |
| 波次头信息 | 系统波次的聚合维度只读；手动波次生成后只读 |
| 波次明细 | 商品与订单信息按出库需求快照展示 |
| 关联进度 | 只读展示下游单据进度，可跳转下游详情 |
| 操作记录 | 按时间倒序展示 |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 分配拣货员 | 状态=草稿 | 选择启用拣货员 |
| 下推拣货 | 状态=草稿且拣货员有效 | 生成/关联 PICK，状态变为拣货中 |
| 作废 | 状态=草稿、拣货中、已拣货、已复核 | 二次确认，作废原因必填 |
| 查看拣货单 | 已生成 PICK | 跳转 PICK 详情 |
| 查看复核/包裹/交运 | 已有关联单据 | 跳转对应下游详情 |

## 6. 测试关注点

- 草稿状态可分配拣货员、下推拣货、作废。
- 下推拣货后状态变为拣货中，不允许重复下推。
- 已交运和已作废为终态，不展示状态变更按钮。
- 作废波次必须二次确认并填写作废原因。
- 关联进度只读展示，不在波次详情页执行拣货、复核、包装、交运。
- 包装完成扣减现存、释放占用的结果可通过下游 PKG/库存流水查看，波次页只展示关联结果。
