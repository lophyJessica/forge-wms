# 波次字段清单

> 角色：字段清单(SSOT) | 类型：业务单据
> 本文件为波次字段唯一事实来源；主 PRD、业务规则、流程推演、Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- 单号、状态、系统聚合结果、系统统计字段不可手工编辑。
- 数量字段按 `11-字段校验规范`：正整数、`>0`、`≤上限值`。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 备注上限 200 字符，空值展示 `-`。
- 商品、承运商、线路、优先级等信息在波次中快照存储，便于追溯当时的出库组织口径。

## 2. 波次头字段

|:--|:--|:--|:--|:--:|:--:|:--|
| waveNo | 波次单号 | string | 系统生成 | 是 | 否 | `WAVE{YYYYMMDD}-{4位序号}`，如 `WAVE20260705-0001` |
| waveType | 波次类型 | enum | 系统聚合/用户选择 | 是 | 否 | `SYSTEM` 系统波次、`MANUAL` 手动波次 |
| status | 波次状态 | enum | 系统 | 是 | 否 | 见“枚举值”；状态只能通过动作按钮触发 |
| warehouseCode | 仓库编码 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 必须为启用仓库；系统波次按出库需求带入 |
| warehouseName | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| carrierCode | 承运商编码 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 系统波次聚合维度之一 |
| carrierName | 承运商名称 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 如顺丰、京东物流；按出库需求快照 |
| routeCode | 线路编码 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 系统波次聚合维度之一 |
| routeName | 线路名称 | string | 系统聚合/用户选择 | 是 | 条件可编辑 | 如上海-浦东线 |
| shipPriority | 发货优先级 | enum | 系统聚合/用户选择 | 是 | 条件可编辑 | `URGENT` 加急、`NORMAL` 普通、`LOW` 低 |
| pickMode | 拣货模式 | enum | 系统推荐/用户选择 | 是 | 是 | `SINGLE_ORDER` 按单拣货、`PICK_SORT` 边拣边分、`BATCH_THEN_SORT` 先拣后播 |
| pickerId | 拣货员ID | string | 用户选择 | 否 | 是 | 草稿状态可分配；必须为启用账号 |
| pickerName | 拣货员 | string | 用户选择 | 否 | 否 | 随拣货员 ID 带出 |
| orderCount | 订单数 | integer | 系统算 | 是 | 否 | `count(distinct demandNo)`，单波次 `≤50` |
| skuCount | SKU 数 | integer | 系统算 | 是 | 否 | `count(distinct productCode)` |
| totalQty | 总件数 | integer | 系统算 | 是 | 否 | `Σ waveQty` |
| plannedPickDate | 计划拣货日期 | date | 系统聚合/用户选择 | 否 | 是 | 不超过当前日期规则不适用；用于作业计划 |
| remark | 备注 | string | 用户输入 | 否 | 是 | ≤200 字符 |

## 3. 波次明细字段（关联的出库需求行）

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| detailId | 明细ID | string | 系统生成 | 是 | 否 | 页面隐藏主键 |
| lineNo | 行号 | integer | 系统生成 | 是 | 否 | 从 1 递增 |
| demandNo | 出库需求单号 | string | ERP SO / WMS 出库需求 | 是 | 否 | 销售订单审核后下发触发波次 |
| soLineNo | SO 行号 | integer | ERP SO | 是 | 否 | 用于追溯销售订单行 |
| receiverArea | 收货区域 | string | ERP SO | 否 | 否 | 用于人工识别线路合理性 |
| productCode | 商品编码 | string | ERP SO / 商品档案 | 是 | 否 | 字母+数字 |
| productName | 商品名称 | string | ERP SO / 商品档案 | 是 | 否 | 商品快照 |
| productSpec | 规格 | string | ERP SO / 商品档案 | 否 | 否 | 无值展示 `-` |
| unit | 单位 | string | ERP SO / 商品档案 | 是 | 否 | 如 件、箱 |
| orderQty | 订单数量 | integer | ERP SO | 是 | 否 | 正整数 `>0` |
| qtyAllocated | 占用数量 | integer | 系统算 | 是 | 否 | SO 审核后占用数量；波次生成不新增占用 |
| waveQty | 波次数量 | integer | 系统算/用户选择 | 是 | 条件可编辑 | 默认等于待出库数量；正整数 `>0`，`≤ orderQty` |
| demandStatus | 出库需求状态 | enum | 系统 | 是 | 否 | `PENDING_WAVE` 待波次、`WAVED` 已入波次、`CANCELLED` 已取消 |

## 5. 枚举值

### 5.1 波次类型 `waveType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| SYSTEM | 系统波次 | 系统按规则自动聚合 |
| MANUAL | 手动波次 | 仓管手动圈选出库需求生成 |

### 5.2 波次状态 status

| 值 | 展示 | 说明 |
|:--|:--|:--|
| DRAFT | 草稿 | 新建或系统定时自动聚合生成，尚未开始拣货作业 |
| PICKING | 拣货中 | 拣货任务已下推，PDA 正在执行或等待执行 |
| PICKED | 已拣货 | 关联的拣货单已全部拣完，等待复核校验 |
| CHECKED | 已复核 | 复核完成且包装称重贴单完毕，等待交运出库 |
| SHIPPED | 已交运 | 包裹已交接给承运商，出库流程完结 |
| VOIDED | 已作废 | 波次异常关闭或被作废，不物理删除 |

### 5.3 发货优先级 `shipPriority`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| URGENT | 加急 | 优先生成和处理 |
| NORMAL | 普通 | 默认优先级 |
| LOW | 低 | 低优先处理 |

### 5.4 拣货模式 `pickMode`

| 值 | 展示 | 适用场景 |
|:--|:--|:--|
| SINGLE_ORDER | 按单拣货 | 大件/异形商品 |
| PICK_SORT | 边拣边分 | 多订单同时拣，PDA 提示分播到发货箱 |
| BATCH_THEN_SORT | 先拣后播 | 大批量同 SKU，集中拣货后统一播种 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 订单数 | `count(distinct demandNo)` | 单波次最多 50 单 |
| SKU 数 | `count(distinct productCode)` | 用于判断拣货复杂度 |
| 总件数 | `Σ waveQty` | 波次作业总数量 |
| 聚合键 | `warehouse + carrier + route + shipPriority` | context 明确承运商、线路、优先级；仓库用于避免跨仓作业 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 实时失焦 | 数量格式、备注长度 |
| 保存/生成 | 仓库、承运商、线路、优先级、明细数量、订单数上限 |
| 下推拣货 | 状态为草稿、明细未取消、未重复入其他未完成波次、拣货员有效 |
| 作废波次 | 状态允许作废、作废原因必填、二次确认 |
