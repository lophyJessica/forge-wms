## 3. 用例 1：系统聚合生成一个波次

### 输入

聚合键：`上海一仓 + 顺丰速运 + 上海-浦东线 + 普通`。

命中出库需求：SO20260705-0101、SO20260705-0102、SO20260705-0103。

### 公式

| 指标 | 公式 | 结果 |
|:--|:--|--:|
| 订单数 | `count(distinct demand_no)` | 3 |
| SKU 数 | `count(distinct sku_code)` | 2 |
| 总件数 | `6 + 4 + 8` | 18 |
| 拆波数量 | `ceil(3 / 50)` | 1 |

### 状态快照

| 时点 | 波次状态 | 出库需求状态 | 库存影响 |
|:--|:--|:--|:--|
| 聚合前 | - | 待波次 | SO 审核已占用 |
| 生成后 | 草稿 | 已入波次 | 不新增占用，不扣减现存 |
| 下推拣货后 | 拣货中 | 拣货中 | 不扣减现存 |
| 包装完成后 | 已复核 | 下游包装完成 | 扣减现存、释放占用，并生成 FL |
| 链路完成后 | 已交运 | 已完成 | 无额外扣减 |

### 期望结果

- 生成 `WAVE20260705-0001`。
- 明细包含 3 个出库需求，订单数 3，SKU 数 2，总件数 18。
- `SO20260705-0104` 因线路不同不进入该波次。
- `SO20260705-0105` 因优先级不同不进入该波次。

## 4. 用例 2：超过 50 单自动拆波

### 输入

同一聚合键下待波次订单数为 126。

### 公式

| 指标 | 公式 | 结果 |
|:--|:--|--:|
| 拆波数量 | `ceil(126 / 50)` | 3 |
| 第 1 波订单数 | `min(50, 126)` | 50 |
| 第 2 波订单数 | `min(50, 76)` | 50 |
| 第 3 波订单数 | `126 - 50 - 50` | 26 |

### 期望结果

| 波次单号 | 订单数 | 状态 |
|:--|--:|:--|
| WAVE20260705-0002 | 50 | 草稿 |
| WAVE20260705-0003 | 50 | 草稿 |
| WAVE20260705-0004 | 26 | 草稿 |

## 5. 用例 3：手动紧急波次

### 输入

仓库主管手动选择 2 单：SO20260705-0105、SO20260705-0110，原因：客服加急。

### 走查

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 打开新增编辑页 | 用户有权限 | 允许进入 |
| 2 | 搜索加急订单 | 状态=待波次 | 可勾选 |
| 3 | 点击生成波次 | 订单数 2≤50 | 生成手动波次 |
| 4 | 分配拣货员 | 拣货员启用 | 可下推拣货 |

### 期望结果

- 生成 `WAVE20260705-0005`，波次类型为手动波次。
- 状态为草稿。
- 备注记录“客服加急”。

## 7. 用例 5：关闭/作废波次

### 输入

| 字段 | 值 |
|:--|:--|
| 波次单号 | WAVE20260705-0006 |
| 当前状态 | 草稿 |
| 关闭原因 | 承运商线路临时停运，等待重新组波 |
| 操作人 | 主管-周敏 |

### 状态快照

| 时点 | 波次状态 | 库存影响 | 说明 |
|:--|:--|:--|:--|
| 关闭前 | 草稿 | 无新增占用，无扣减 | SO 审核占用仍按订单规则存在 |
| 关闭后 | 已作废 | 不自动释放库存 | 是否释放取决于订单取消/重组规则 |

## 8. 走查清单

| 检查项 | 预期 |
|:--|:--|
| WAVE 前缀 | 已在 `09-单据命名规则` 定义 |
| 聚合维度 | 承运商、线路、发货优先级；仓库用于避免跨仓 |
| 50 单上限 | 系统波次和手动波次都生效 |
| 状态按钮 | 状态只能由动作按钮改变 |
| 库存占用 | SO 审核占用，波次不新增占用 |
| 库存扣减 | 包装完成扣减现存、释放占用并生成 FL |
| 下游边界 | 不把拣货/复核/包装执行细节写入波次 |
