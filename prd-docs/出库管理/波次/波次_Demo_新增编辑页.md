# 波次_Demo_新增编辑页

> 角色：Demo·新增编辑页 | 类型：业务单据
> 本页用于手动波次生成；系统波次由定时任务聚合，本页只提供 Demo 中的人工触发和圈选能力。

## 1. 页面定位

新增编辑页用于仓库主管从待波次出库需求池中圈选订单，生成手动波次。适用于紧急订单、异常处理、客服加急等需要人工优先组织的场景。

## 2. 页面结构

```text
新建手动波次
├─ 基础信息区
│  ├─ 仓库、承运商、线路、发货优先级
│  ├─ 拣货模式、计划拣货日期、拣货员
│  └─ 备注
├─ 待选出库需求区
│  ├─ 筛选：SO单号、SKU、承运商、线路、优先级、收货区域
│  ├─ 待选表格
│  └─ 勾选/全选当前页
├─ 已选明细区
│  ├─ 已选订单数、SKU数、总件数
│  ├─ 已选出库需求明细
│  └─ 移除
└─ 底部动作栏
   ├─ 取消
   ├─ 保存草稿式选择（仅页面暂存，不生成波次）
   └─ 生成波次
```

## 3. 控件说明

| 控件 | 说明 |
|:--|:--|
| 仓库 | 必选，过滤待选出库需求 |
| 承运商 | 必选，建议与已选需求保持一致 |
| 线路 | 必选，建议与已选需求保持一致 |
| 发货优先级 | 必选，用于人工组波口径 |
| 拣货模式 | 必选，默认可由系统根据 SKU/订单结构推荐 |
| 拣货员 | 可选；也可生成后在列表分配 |
| 待选表格 | 仅展示状态为待波次的出库需求 |
| 生成波次 | 全量校验通过后生成 WAVE，状态为草稿 |

## 4. 交互规则

| 场景 | 规则 |
|:--|:--|
| 勾选出库需求 | 加入已选明细，实时刷新订单数、SKU数、总件数 |
| 已选订单数超过 50 | 阻断生成，提示拆分波次 |
| 选择已取消需求 | 不允许勾选，或生成时阻断 |
| 选择已入未完成波次需求 | 阻断加入，提示已在其他波次中 |
| 点击取消 | 返回列表，不生成波次 |
| 点击生成波次 | 生成 `WAVE{YYYYMMDD}-{4位序号}`，跳转详情页 |

## 6. 校验提示

| 场景 | 文案 |
|:--|:--|
| 未选择出库需求 | `请至少选择一条待波次出库需求` |
| 超过 50 单 | `单个波次最多 50 单，请拆分后再生成` |
| 需求已入波次 | `出库需求已在未完成波次中` |
| 备注超长 | `备注不能超过 200 字符` |
| 拣货员无效 | `请选择启用状态的拣货员` |
