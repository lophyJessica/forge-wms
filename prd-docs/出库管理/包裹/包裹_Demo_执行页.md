# 包裹_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页覆盖称重、贴面单、生成包裹号、确认包装完成和库存过账交互；字段定义以 `包裹字段清单.md` 为准。

## 1. 页面定位

包裹执行页用于包装员处理由 CHECK 复核通过下推的 PKG，完成“称重 → 生成/录入面单 → 打印并贴面单 → 确认包装完成”。确认包装完成是库存实扣动作入口，页面需要对库存过账结果给出清晰反馈。

## 2. 页面结构

```text
包裹包装执行页
├─ 任务头
│  ├─ 包裹号、来源复核单、状态、过账状态
│  └─ 仓库、包装员、包装台、承运商
├─ 包装信息区
│  ├─ 包裹重量
│  ├─ 长/宽/高
│  └─ 出库总数
├─ 面单区
│  ├─ 承运商
│  ├─ 面单号
│  ├─ 生成/录入面单
│  ├─ 打印面单
│  └─ 确认已贴单
├─ 明细区
│  ├─ 商品、货位、出库数量
│  └─ 库存快照：现存、占用、可用
├─ 过账结果区
│  ├─ FL 单号
│  ├─ 过账前后数量
│  └─ 失败原因
└─ 底部动作区
   ├─ 保存包装信息
   └─ 确认包装完成
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 开始包装 | 待包装任务进入包装中，写入包装员和包装台 |
| 称重 | 录入或读取重量，重量必须大于 0 |
| 选择承运商 | 默认继承上游承运商，可按权限确认调整 |
| 生成/录入面单 | Demo 版支持模拟生成或人工录入 |
| 打印面单 | 面单号存在后允许打印，成功后状态为已打印 |
| 确认已贴单 | 包装员确认面单已贴到实物包裹 |
| 确认包装完成 | 校验包装资料，执行库存过账，生成 FL，流转交运 |

## 4. 校验反馈

| 场景 | 反馈 |
|:--|:--|
| 重量为空 | 重量输入框标红，提示“请先完成包裹称重” |
| 承运商为空 | 承运商选择框标红，提示“请选择承运商” |
| 面单为空 | 提示“请先生成或录入面单号” |
| 未打印面单 | 提示“请先打印面单” |
| 未确认贴单 | 提示“请确认面单已贴到包裹” |
| 库存校验失败 | 过账结果区展示失败原因，PKG 保持包装中 |
| 包装完成成功 | 展示 FL 单号和“已流转交运” |

## 5. 页面状态

| PKG 状态 | 页面表现 |
|:--|:--|
| 待包装 | 展示开始包装按钮 |
| 包装中 | 展示包装信息、面单、明细和确认包装完成 |
| 已包装 | 只读展示包装摘要、库存过账结果和 FL 单号 |
| 已作废 | 只读展示“任务已作废”，不允许包装 |

## 8. 离线与同步

- 包装执行页可离线保存称重、面单和贴单草稿。
- 确认包装完成必须以后端在线库存校验为准；离线状态不允许完成库存过账。
- 网络恢复后同步包装草稿，仍需重新点击确认包装完成。
- 已包装状态同步后只读，不允许再次提交。

## 9. 不展示内容

- 不展示新增包裹入口。
- 不展示审核按钮。
- 不展示复核、拣货、交运确认控件。
- 不展示手工修改库存三口径或手工生成 FL 的入口。
