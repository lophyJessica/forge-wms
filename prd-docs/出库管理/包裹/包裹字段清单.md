# 包裹字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为包裹字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- PKG 单号、来源 CHECK、来源 PICK、来源 WAVE、商品快照、出库数量不可手工编辑。
- 重量、承运商、面单号、面单打印状态是确认包装完成前的关键校验字段。
- 包装完成后，PKG 明细、重量、面单、承运商、库存过账结果均只读。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 空值展示 `-`，备注/异常说明 ≤200 字符。
- 库存三口径字段只记录包装完成前后快照，不允许人工编辑。

## 2. 包裹头字段（承运商·重量·面单·过账）

| sourceCheckNo | 来源复核单号 | string | CHECK 下推 | 是 | 否 | 只读，PKG 必须关联已复核 CHECK |

| sourceWaveNo | 来源波次单号 | string | 继承 CHECK | 是 | 否 | 追溯 WAVE |
| warehouseCode | 仓库编码 | string | 继承 CHECK | 是 | 否 | 必须为启用仓库 |
| warehouseName | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| boxNo | 发货箱/包厢号 | string | 继承 CHECK | 是 | 否 | 已复核通过的箱码 |
| carrierCode | 承运商编码 | string | 继承波次/人工确认 | 是 | 条件是 | 必须为启用承运商；包装完成后只读 |
| carrierName | 承运商名称 | string | 承运商档案 | 是 | 否 | 随承运商编码带出 |
| serviceType | 服务类型 | enum | 继承/人工确认 | 否 | 条件是 | 如 `STANDARD` 标准、`EXPRESS` 加急 |
| packageWeightKg | 包裹重量(kg) | decimal | 电子秤/人工录入 | 是 | 是 | `>0`，建议保留 3 位小数 |
| packageLengthCm | 长(cm) | decimal | 人工录入 | 否 | 是 | `>0`，可为空 |
| packageWidthCm | 宽(cm) | decimal | 人工录入 | 否 | 是 | `>0`，可为空 |

| waybillPrintStatus | 面单打印状态 | enum | 系统/打印动作 | 是 | 否 | `NOT_PRINTED` 未打印、`PRINTED` 已打印、`FAILED` 打印失败 |
| labelAttached | 是否已贴面单 | boolean | 包装员确认 | 是 | 是 | 包装完成前必须为 true |

| postStatus | 库存过账状态 | enum | 系统 | 是 | 否 | `NOT_POSTED` 未过账、`POSTED` 已过账、`POST_FAILED` 过账失败 |
| flNo | 库存流水单号 | string | 包装完成生成 | 否 | 否 | `FL{YYYYMMDD}-{8位序号}`，未过账展示 `-` |
| totalOutboundQty | 出库总数 | integer | 系统算 | 是 | 否 | `Σ outboundQty` |
| packerId | 包装员ID | string | 当前用户 | 否 | 否 | 开始包装后写入 |
| packerName | 包装员 | string | 用户档案 | 否 | 否 | 随包装员 ID 带出 |
| packStationCode | 包装台编码 | string | 工作站配置 | 否 | 否 | 如 `WS-PACK-01` |

| packedAt | 包装完成时间 | datetime | 系统 | 否 | 否 | 确认包装完成成功后写入 |

## 5. 枚举值

### 5.1 包裹状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING_PACK | 待包装 | CHECK 已复核，等待包装 |
| PACKING | 包装中 | 已开始称重、面单或贴单作业 |
| PACKED | 已包装 | 包装完成，库存已过账，FL 已生成 |
| VOIDED | 已作废 | 上游异常关闭且未包装完成 |

### 5.2 库存过账状态 `postStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| NOT_POSTED | 未过账 | 尚未确认包装完成 |
| POSTED | 已过账 | 现存和占用已更新，FL 已生成 |
| POST_FAILED | 过账失败 | 库存校验或 FL 生成失败，PKG 不得完成 |

### 5.3 面单打印状态 `waybillPrintStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| NOT_PRINTED | 未打印 | 尚未打印面单 |
| PRINTED | 已打印 | 已完成面单打印 |
| FAILED | 打印失败 | 打印失败，需重打 |

### 5.4 服务类型 `serviceType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| STANDARD | 标准 | 普通时效 |
| EXPRESS | 加急 | 优先交运 |
| ECONOMY | 经济 | 低成本配送 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 出库总数 | `Σ outboundQty` | 包裹内商品出库数量合计 |
| 已过账数量 | `Σ postedQty` | 包装完成后应等于出库总数 |
| 过账后现存 | `过账前现存 - 出库数量` | 明细级库存实扣 |
| 过账后占用 | `过账前占用 - 出库数量` | 明细级释放占用 |
| 过账后可用 | `过账前可用` | 包装完成不再改变可用 |
| 完成条件 | `重量>0 且承运商存在 且面单号存在 且面单已打印且已贴单` | 满足后才进入库存过账 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 开始包装 | 来源 CHECK 是否已复核，PKG 是否待包装 |
| 重量录入 | 重量是否大于 0，格式是否为数字 |
| 面单生成/录入 | 面单号是否为空，是否与当前 PKG 绑定 |
| 打印面单 | 面单号存在且打印状态可更新 |
| 确认包装完成 | 重量、承运商、面单、贴单、明细、库存快照、幂等校验 |
| 库存过账 | 现存和占用是否足以覆盖出库数量，FL 单号是否可生成 |
