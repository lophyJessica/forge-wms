# 包裹_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看包裹来源、包装信息、出库明细、库存过账结果、FL 和下游交运关联。

## 1. 页面定位

包裹详情页面向研发、测试、产品复核，用于核对 PKG 是否由 CHECK 复核通过下推、包装资料是否完整、确认包装完成时库存三口径是否正确变化，并确认下游交运不再产生库存过账。

## 2. 页面结构

```text
包裹详情
├─ 顶部摘要
│  ├─ 包裹号、状态、过账状态、FL 单号
│  └─ 返回列表、查看来源复核单、查看交运单、查看 FL
├─ 来源信息
│  ├─ 来源复核单、来源拣货单、来源波次
│  └─ 仓库、发货箱/包厢
├─ 包装信息
│  ├─ 包装员、包装台、重量、尺寸
│  └─ 承运商、面单号、打印状态、贴单状态
├─ 包裹明细
│  ├─ 商品、货位、批次、出库数量
│  └─ 过账前后现存、占用、可用
├─ 库存流水
│  └─ FL 单号、变动类型、变动数量、变动后现存、过账时间
├─ 关联进度
│  └─ 下游 DSH 单号和状态
└─ 操作记录
   └─ 生成、开始包装、称重、面单、贴单、包装完成、过账、流转交运
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态只展示 PKG 自身状态：待包装、包装中、已包装、已作废 |
| 来源信息 | 来源 CHECK、PICK、WAVE 只读，可跳转详情 |
| 包装信息 | 重量保留 3 位小数；空值展示 `-` |
| 包裹明细 | 数量右对齐；过账前后库存快照只读 |
| 库存流水 | 已包装时展示 FL；未过账时展示 `-` |
| 关联进度 | 只读展示 DSH，不在本页执行交运确认 |
| 库存说明 | 展示“包装完成时现存减少、占用减少、可用保持不变；交运不再过账” |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 查看来源复核单 | 有 source_check_no | 跳转 CHECK 详情 |
| 查看来源拣货单 | 有 source_pick_no | 跳转 PICK 详情 |
| 查看来源波次 | 有 source_wave_no | 跳转 WAVE 详情 |
| 查看 FL | 有 fl_no | 跳转库存流水详情 |
| 查看交运单 | 有 dsh_no | 跳转 DSH 详情 |
| 进入包装执行 | 状态=待包装/包装中且有权限 | 跳转执行页 |
| 返回列表 | 全部状态 | 返回包裹列表 |

> 详情页不展示新增、审核、复核、拣货、交运确认或手工库存调整操作。

## 7. 测试关注点

- PKG 详情能追溯来源 CHECK、PICK、WAVE。
- 状态不出现审核流状态。
- 未包装完成时不展示 FL。
- 已包装后库存快照正确：现存减少、占用减少、可用保持不变。
- 已包装后重复提交不会再次影响库存。
- 下游交运只做交接，不再产生库存过账。
