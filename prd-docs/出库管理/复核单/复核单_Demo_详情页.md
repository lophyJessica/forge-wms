# 复核单_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看复核单来源、明细、扫码记录、差异退回和下游包裹关联。

## 1. 页面定位

复核单详情页面向研发、测试、产品复核，用于核对 CHECK 是否由 PICK 下推、箱码/商品扫码是否留下记录、应复核数和实复核数是否一致，并确认 CHECK 阶段没有发生库存过账。

## 2. 页面结构

```text
复核单详情
├─ 顶部摘要
│  ├─ 复核单号、状态、复核结果、异常标记
│  └─ 返回列表、查看来源拣货单、查看包裹
├─ 复核头信息
│  ├─ 来源拣货单、来源波次、仓库、复核员、复核端
│  └─ 发货箱/包厢、应复核总数、实复核总数、差异总数、差异行数
├─ 复核明细
│  ├─ 发货箱、商品、应复核数、实复核数、差异数量、差异类型
│  └─ 明细状态、完成时间、行备注
├─ 扫码记录
│  ├─ 扫箱记录
│  ├─ 扫商品记录
│  └─ 错扫/差异记录
├─ 退回记录
│  └─ 退回原因、退回说明、退回人、退回时间
├─ 关联进度
│  └─ 来源 PICK 状态、下游 PKG 单号和状态
└─ 操作记录
   └─ 生成、开始、扫码、通过、退回、作废
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态只展示 CHECK 自身状态：待复核、复核中、已复核、已退回拣货、已作废 |
| 来源信息 | 来源 PICK 和 WAVE 只读，可跳转详情 |
| 复核明细 | 数量右对齐；差异数量不为 0 的行标记异常 |
| 扫码记录 | 展示成功记录和阻断记录，便于排查错扫/串箱 |
| 退回记录 | 仅已退回拣货或存在退回历史时展示 |
| 关联进度 | 只读展示 PKG，不在本页执行包装 |
| 库存说明 | 展示“CHECK 阶段不扣现存、不释放占用、不生成 FL；包装完成才实扣” |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 查看来源拣货单 | 有 source_pick_no | 跳转 PICK 详情 |
| 查看来源波次 | 有 source_wave_no | 跳转 WAVE 详情 |
| 查看包裹 | 有 pkg_no | 跳转 PKG 详情 |
| 进入复核执行 | 状态=待复核/复核中且有权限 | 跳转执行页 |
| 返回列表 | 全部状态 | 返回复核单列表 |

> 详情页不展示新增、审核、反审核、包装完成、交运确认、库存过账操作。

## 7. 测试关注点

- CHECK 详情能追溯来源 PICK 和 WAVE。
- 状态不出现待审核、已审核等审核流状态。
- 明细能展示应复核数、实复核数、差异数量和差异类型。
- 错扫箱、错扫商品、少件、多件都有扫码/操作记录。
- CHECK 已复核后只读，不允许继续修改实复核数量。
- CHECK 阶段不生成 FL；库存扣减结果只能在下游包装完成后的库存流水中查看。
