## 8. 用例 5：多件/超量复核

### 输入

| 字段 | 值 |
|:--|:--|
| 复核单 | CHECK20260705-0003 |
| SKU | SKU005 白雪直液式走珠笔红色 |
| 应复核数 | 5 |
| 实复核数 | 6 |

### 公式

| 字段 | 公式 | 结果 |
|:--|:--|--:|
| 差异数量 | `6 - 5` | +1 |
| 差异类型 | `actual > should` | over |

### 期望结果

- 系统提示 `复核数量超出，不能通过`。
- CHECK 不能进入已复核。
- 复核员选择“确认复核不通过”，退回拣货核对是否多拣、错拣或串箱。
- 不扣现存、不释放占用、不生成 FL。

## 9. 用例 6：复核通过流转包装

| 条件 | 判断 | 期望结果 |
|:--|:--|:--|
| 所有明细 `line_status=matched` 且无错品/串箱 | 通过条件满足 | CHECK 状态=已复核 |
| CHECK 状态=已复核 | 下游流转条件满足 | 生成/流转 PKG 待包装任务 |
| PKG 尚未包装完成 | 库存实扣条件不满足 | 现存、占用、可用均不变 |

## 10. 用例 7：复核不通过后重新复核

| 步骤 | 输入/动作 | 期望结果 |
|:--:|:--|:--|
| 1 | CHECK20260705-0002 少件，确认不通过 | CHECK 状态=已退回拣货 |
| 2 | 关联 PICK 回到重拣/补拣处理 | 拣货员补齐缺失商品 |
| 3 | 重拣完成后再次进入复核 | 生成或恢复待复核任务，具体形态 context 未明确 |
| 4 | 新一轮复核数量一致 | 才允许流转 PKG |

## 11. 走查清单

| 检查项 | 预期 |
|:--|:--|
| CHECK 前缀 | 已在 `09-单据命名规则` 定义 |
| 来源限制 | 只能由 PICK 已拣完下推，不提供新增入口 |
| 状态 | 仅待复核、复核中、已复核、已退回拣货、已作废 |
| 扫码顺序 | 先扫发货箱/包厢，再扫商品 |
| 扫错箱 | 阻断并反馈 |
| 扫错商品 | 记录异常，不允许通过 |
| 数量不符 | 不允许通过，必须回退拣货 |
| 库存过账 | 复核不扣现存、不释放占用、不生成 FL；包装完成才实扣 |
