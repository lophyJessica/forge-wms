# 复核单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页覆盖 PDA/工作站复核作业，重点描述扫箱、扫商品、数量校验和退回拣货交互；字段定义以 `复核单字段清单.md` 为准。

## 1. 页面定位

复核执行页用于复核员对 CHECK 执行“扫描发货箱/包厢 → 逐件扫描商品 → 校验数量 → 确认通过/不通过”。PDA 端适合移动复核，工作站端适合固定复核台连续扫码；两端共用同一套业务规则。

## 2. 页面结构

```text
复核执行页（PDA/工作站）
├─ 任务头
│  ├─ 复核单号、来源拣货单、进度
│  └─ 仓库、复核员、复核端、发货箱/包厢
├─ 扫箱区
│  ├─ 发货箱/包厢扫码输入
│  └─ 箱码校验结果
├─ 扫商品区
│  ├─ 商品扫码输入
│  ├─ 最近扫描商品
│  └─ 声音/震动/颜色反馈
├─ 明细进度区
│  ├─ SKU、商品、应复核数、实复核数、待扫描数、差异
│  └─ 明细状态：待复核/复核中/已匹配/有差异
├─ 差异处理区
│  ├─ 差异摘要
│  ├─ 退回原因
│  └─ 退回说明
└─ 底部动作区
   ├─ 确认复核通过
   └─ 确认复核不通过
```

## 3. PDA 与工作站差异

| 端 | 交互重点 | 说明 |
|:--|:--|:--|
| PDA | 大字体、大按钮、扫码优先、震动反馈 | 适合移动到发货箱旁复核 |
| 工作站 | 光标自动聚焦扫码框、键盘快捷确认、连续扫码列表 | 适合固定复核台高频扫码 |

两端都不提供新增 CHECK、审核、包装完成、库存扣减操作。

## 4. 核心交互

| 交互 | 说明 |
|:--|:--|
| 开始复核 | 待复核任务进入复核中，写入复核员和开始时间 |
| 扫发货箱/包厢 | 箱码匹配当前 CHECK 后才允许扫商品 |
| 扫商品 | 商品属于当前箱/订单时累计实复核数 |
| 数量自动累计 | 每次成功扫码 +1，实时刷新进度和待扫描数量 |
| 多件提示 | 实复核数超过应复核数时标记多件差异 |
| 错品提示 | 扫到非当前明细商品时标记错品异常 |
| 确认复核通过 | 仅全部明细已匹配时展示 |
| 确认复核不通过 | 存在已确认差异，或复核员将未匹配数量确认为少件时展示，必须选择退回原因 |

## 5. 扫码反馈

| 场景 | 反馈 |
|:--|:--|
| 扫箱正确 | 箱码区域高亮，短提示音 |
| 扫错箱 | 红色提示、震动、语音提示“箱码不属于当前复核单” |
| 扫商品正确 | 明细行数量 +1，高亮最近扫描商品 |
| 扫错商品 | 红色提示、震动、语音提示“商品不属于当前复核单” |
| 待扫描未完成 | 待扫描数量大于 0，不展示确认通过 |
| 多件 | 差异栏展示正数，提示“数量已超过应复核数” |
| 退回原因未选 | 阻断提交，提示“请选择退回原因” |

## 6. 页面状态

| CHECK 状态 | 页面表现 |
|:--|:--|
| 待复核 | 展示开始复核按钮 |
| 复核中 | 展示扫箱区、扫商品区、明细进度和差异处理 |
| 已复核 | 只读展示完成摘要，不允许继续扫码 |
| 已退回拣货 | 只读展示退回原因和差异明细，不允许继续扫码 |
| 已作废 | 只读展示“任务已作废”，显示“该任务已被作废或上游已关闭” |

## 9. 离线与同步

- PDA 可离线缓存扫描记录，网络恢复后自动同步。
- 离线期间可做本地基础校验：箱码、商品、扫描数量。
- 同步时以后端最新 CHECK 状态为准；若 CHECK 已复核、已退回或已作废，提示重新同步任务。
- 离线记录不得绕过复核通过条件；存在差异时仍不能确认通过。

## 10. 不展示内容

- 不展示新增复核单入口。
- 不展示待审核、已审核、反审核按钮。
- 不展示包装完成、交运确认、库存扣减、释放占用、生成 FL 操作。
