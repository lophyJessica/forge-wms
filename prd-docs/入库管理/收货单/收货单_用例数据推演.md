## 3. 用例 1：正常收货并打印标签

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260705-0001 |
| 来源采购单号 | PO20260705-0008 |
| 仓库/库区 | 上海一仓 / 收货区 |
| SKU003 本次实收 | 60 包 |
| SKU001 本次实收 | 50 盒 |
| 收货日期 | 2026-07-05 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | PO 下推生成 RCV | PO 有效 | RCV 状态为待收货 |
| 2 | 仓管录入实收 | 60≤60，50≤50 | 实时校验通过，状态变为收货中 |
| 3 | 点击确认收货 | 必填、数量、超收校验通过 | RCV 状态变为待质检，库存记入冻结 |
| 4 | 打印标签 | 待质检且条码存在 | 打印次数 0→1 |

### 期望结果

- RCV20260705-0001 状态为待质检。
- 标签条码生成，示例：`RCV20260705-0001-01`、`RCV20260705-0001-02`。
- PO20260705-0008 两行未收数量变为 0。
- 后续可进入质检登记阶段。

## 4. 用例 2：部分收货

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260706-0001 |
| 来源采购单号 | PO20260706-0003 |
| SKU006 采购数量 | 30 个 |
| SKU006 历史已收 | 0 个 |
| SKU006 本次实收 | 18 个 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 录入本次实收 18 | 18≤30 | 通过，RCV 状态变为收货中 |
| 2 | 点击确认收货 | 数量为正整数，未超收 | RCV 状态变为待质检，库存记入冻结 |
| 3 | 更新 PO 收货进度 | 0+18 | 历史已收变为 18，未收保留 12 |

### 期望结果

- 部分收货允许确认。
- PO 未收余量 12 箱保留，收货单 RCV20260706-0001 状态转为待质检。

## 5. 用例 3：超收阻断

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260705-0002 |
| 来源采购单号 | PO20260705-0008 |
| SKU003 PO 未收 | 60 包 |
| SKU003 本次实收 | 65 包 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 录入本次实收 65 | 65>60 | 字段标红 |
| 2 | 点击确认收货 | 超收校验失败 | 阻断确认 |
| 3 | 仓管改为 60 | 60≤60 | 可重新确认收货 |

### 期望结果

- 系统提示：`本次实收数量不能大于 PO 未收数量`。
- RCV 保持待收货/收货中。
- 不生成正式收货标签，无法进入质检登记。

## 6. 用例 4：质检全部不合格转异常

### 输入

| 字段 | 值 |
|:--|:--|
| 收货单号 | RCV20260705-0003 |
| SKU | SKU001 双鸭牌标准型回形针 |
| 本次实收 | 20 卷 |
| 质检登记 - 合格数量 | 0 卷 |
| 质检登记 - 不合格数量 | 20 卷 |
| 质检登记 - 不合格原因 | 外箱受潮 |

### 推演

| 步骤 | 动作 | 系统判断 | 结果 |
|:--:|:--|:--|:--|
| 1 | 确认收货 | 收货校验通过 | RCV 状态变为待质检，库存记入冻结 |
| 2 | 进行质检登记 | 校验 `0 + 20 = 20`，且合格数量 = 0 | RCV 状态变为异常，不生成上架单 |
| 3 | 尝试流转上架 | 处于异常状态且无合格品 | 阻断，无法生成上架单 PUT |

### 期望结果

- RCV20260705-0003 状态变为异常。
- 不生成上架单 PUT，不允许上架。
- 等待退货处理；一期只登记不合格数量和原因，完整退货为二期。
