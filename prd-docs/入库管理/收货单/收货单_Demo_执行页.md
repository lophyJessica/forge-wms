# 收货单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页覆盖 PC 收货作业页与 PDA 快速收货作业页；两端共用同一套 RCV 字段、状态和超收校验规则。

## 1. 页面定位

执行页用于仓管基于 PO 下推的 RCV 待收货单完成收货录入、保存草稿、确认收货、打印标签，以及在确认后对 RCV 进行质检结果登记（合格/不合格数量和原因）。PC 适合完整核对、批量录入和质检登记；PDA 仅用作收货现场扫码、少量改数和快速确认。

## 2. 页面结构

### 2.1 PC 收货与质检作业页

```text
PC 收货与质检执行页
├─ 顶部信息条
│  ├─ 收货单号、状态、来源PO、供应商
│  └─ 返回列表、查看详情
├─ 基础信息区
│  ├─ 仓库、库区、收货日期
│  └─ 备注
├─ 商品明细区
│  ├─ PO商品快照、采购数量、已收数量、PO未收数量
│  ├─ [收货状态下可编辑] 本次实收数量、行备注
│  └─ [待质检状态下可编辑] 合格数量、不合格数量（计算）、不合格原因、行备注
├─ 校验提示区
│  └─ 超收/必填/格式错误/质检守恒校验等字段级提示
└─ 底部动作栏
   ├─ 保存草稿（仅待收货/收货中状态）
   ├─ 确认收货（仅待收货/收货中状态）
   ├─ 提交质检登记（仅待质检状态）
   └─ 打印标签（待质检及后续状态展示）
```

### 2.2 PDA 快速收货作业页

```text
PDA 收货作业页
├─ 扫码区
│  ├─ 扫 PO/RCV 条码
│  └─ 扫商品条码
├─ 当前商品卡片
│  ├─ 商品名称、规格、单位
│  └─ PO未收数量、本次已录数量
├─ 数量录入区
│  ├─ 本次实收数量
│  └─ 加减步进、手动输入
├─ 反馈区
│  └─ 超收提示、语音+震动反馈
└─ 底部大按钮
   ├─ 保存
   └─ 确认收货
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 仓库选择 | 默认带出常用仓库，可切换启用仓库（仅待收货/收货中可改） |
| 库区选择 | 默认收货区，列表受所选仓库过滤（仅待收货/收货中可改） |
| 实收录入 | 失焦校验正整数和 PO 未收上限 |
| 保存草稿 | 保存当前录入的实收数量，状态变为“收货中”或保持不变 |
| 确认收货 | 全量严格校验，状态变为“待质检”，数量锁定并记入冻结 |
| 打印标签 | 待质检及后续状态展示，打印成功后更新打印次数 |
| 质检登记录入 | 仅“待质检”状态下展示合格数量与不合格原因录入，失焦校验合格数 ≤ 实收数 |
| 提交质检登记 | 状态变为“待上架”（若合格数 > 0 且自动生成 PUT）或“异常”（若合格数 = 0） |

## 4. 字段状态

| 页面状态 | 可编辑 | 只读 |
|:--|:--|:--|
| 待收货 / 收货中 | 仓库、库区、收货日期、本次实收数量、备注 | 收货单号、PO、供应商、商品快照、采购数量、已收数量、PO 未收数量、状态 |
| 待质检 | 每一行的合格数量、不合格原因、行备注 | 收货单号、PO、供应商、商品快照、采购数量、已收数量、PO 未收数量、本次实收数量、状态、仓库/库区 |
| 待上架 / 已完成 / 异常 / 已作废 | 无 | 全部业务字段及质检结果只读；可打印标签 |

## 5. 校验提示

| 场景 | 提示位置 | 文案 |
|:--|:--|:--|
| 实收为空 | 明细行本次实收数量 | `本次实收数量不能为空` |
| 实收非正整数 | 明细行本次实收数量 | `本次实收数量必须为大于 0 的整数` |
| 超收 | 明细行本次实收数量 | `本次实收数量不能大于 PO 未收数量` |
| 质检数量不匹配 | 明细行合格/不合格数量 | `合格数量与不合格数量之和必须等于本次实收数量` |
| 缺失不合格原因 | 明细行不合格原因 | `有不合格数量时必须选择不合格原因` |
| 备注超长 | 备注输入框 | `备注不能超过 200 字符` |
| 重复操作 | 页面顶部提示 | `当前状态下不可重复执行此操作` |

## 6. 操作后反馈

| 动作 | 成功反馈 | 后续 |
|:--|:--|:--|
| 保存草稿 | `草稿已保存` | 停留当前页 |
| 确认收货 | `收货已确认` | 页面切换为只读并展示质检登记表项与打印标签按钮，状态变为“待质检” |
| 提交质检登记 | `质检登记成功` | RCV 状态变为“待上架”或“异常”，页面切换为完全只读并展示跳转上架单入口 |
| 打印标签 | `标签已发送打印` | 更新打印次数和最后打印时间 |

## 8. PC/PDA 差异

| 项 | PC | PDA |
|:--|:--|:--|
| 使用场景 | 办公区核对 PO、批量录入、打印标签、质检登记 | 收货现场扫码、快速录入、即时反馈，不支持质检登记 |
| 输入方式 | 表格录入为主 | 扫码优先，数量用大按钮/步进器 |
| 信息密度 | 展示完整单据头、多行明细及质检编辑区 | 聚焦当前 RCV 和当前 SKU，仅支持收货录入 |
| 校验 | 失焦校验 + 确认全量校验 | 扫码/录数即时校验 + 确认全量校验 |
| 反馈 | 字段标红、顶部提示 | 字段高亮、语音+震动反馈 |
| 离线 | 不要求 | 可自动切换离线，网络恢复后自动同步；同步前仍不得绕过超收规则 |

PDA 与 PC 不能同时确认同一张草稿/待收货 RCV；确认时以后端最新状态为准，若另一端已操作，则当前端提示“当前状态下不可重复执行此操作”。
