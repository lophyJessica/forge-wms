# 收货单_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看收货单快照、收货结果、标签打印记录和后续质检/上架关联进度。

## 1. 页面定位

收货单详情页面向研发、测试、产品复核，用于确认一张 RCV 的完整只读信息：来源 PO、供应商、仓库/库区、实收明细、状态、标签、后续质检和上架关联结果。

## 2. 页面结构

```text
收货单详情
├─ 顶部摘要
│  ├─ 收货单号、状态
│  └─ 返回列表、打印标签、执行收货、质检登记、作废
├─ 来源信息
│  ├─ 来源采购单号、供应商
│  └─ 创建人/创建时间、确认人/确认时间
├─ 收货信息
│  ├─ 仓库、库区、收货日期、备注
│  └─ 标签条码、打印次数、最后打印时间
├─ 商品明细
│  └─ 商品快照、采购数量、已收数量、PO未收数量、本次实收数量、合格数量、不合格数量、不合格原因
├─ 关联去向
│  └─ 上架单号、上架进度、库存流水入口
└─ 操作记录
   └─ 创建、保存、确认、打印标签、质检登记、作废
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态展示收货单自身状态：待收货、收货中、待质检、待上架、异常、已完成、已作废 |
| 来源信息 | PO、供应商只读，展示 PO 下发快照 |
| 收货信息 | 确认收货后展示确认人和确认时间；空值展示 `-` |
| 商品明细 | 数量字段右对齐；不合格数量大于0时标红展示 |
| 关联去向 | 未生成上架单时展示 `-` |
| 操作记录 | 按时间倒序展示 |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 打印标签 | 状态为 待质检、待上架、已完成、异常 | 打印收货标签并记录次数 |
| 进入执行页/收货 | 状态为 待收货、收货中 | 从详情进入收货执行页继续录入 |
| 质检登记 | 状态为 待质检 | 点击进入质检登记页录入质检结果 |
| 查看上架单 | 已生成 PUT | 跳转上架详情 |

## 5. 关联上架进度

| 状态/上架进度 | 判断逻辑 | 页面说明 |
|:--|:--|:--|
| 待上架 | 已生成 PUT 但累计上架数量 = 0 | 可跳转关联上架单 |
| 上架中 | 0 < 累计上架数量 < 合格总数 | 可跳转关联上架单并查看即时过账流水 |
| 已入库/已完成 | 累计上架数量 = 合格总数 | 可查看全部上架对应的库存流水 |

## 7. 测试关注点

- 待收货/收货中详情页可进入执行页录入，其它状态详情页只读。
- 待质检及后续状态支持打印标签，待收货/收货中不可打印正式标签。
- 质检全部不合格转异常时，详情页状态显示为“异常”且不展示上架单号和上架入口。
- 空值统一展示 `-`，时间格式为 `YYYY-MM-DD HH:mm:ss`。
