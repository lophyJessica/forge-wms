# 收货单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为收货单字段唯一事实来源；主 PRD、业务规则、Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- 单号、状态、来源 PO、供应商、商品快照字段不可手工编辑。
- 数量字段按 `11-字段校验规范`：正整数、`>0`、`≤上限值`。
- 备注字段上限 200 字符。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 空值展示 `-`。
- 商品信息在单据中快照存储，后续主数据变化不影响历史收货单。

| supplierCode | 供应商编码 | string | 继承 PO | 是 | 否 | 供应商快照字段 |
| supplierName | 供应商名称 | string | 继承 PO | 是 | 否 | 供应商快照字段 |
| warehouseCode | 仓库编码 | string | 仓管选择/默认仓库 | 是 | 是 | 必须为启用状态仓库 |
| warehouseName | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| zoneCode | 库区编码 | string | 仓管选择 | 是 | 是 | 默认收货区；必须属于所选仓库 |
| zoneName | 库区名称 | string | 库区档案 | 是 | 否 | 随库区编码带出 |

| createdBy | 创建人 | string | 系统 | 是 | 否 | 当前操作人或 PO 下推任务 |
| createdAt | 创建时间 | datetime | 系统 | 是 | 否 | `YYYY-MM-DD HH:mm:ss` |
| confirmedBy | 确认人 | string | 系统 | 否 | 否 | 点击确认收货的操作人 |
| confirmedAt | 确认时间 | datetime | 系统 | 否 | 否 | 已确认后写入 |

## 3. 明细行字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| lineNo | 行号 | integer | 系统 | 是 | 否 | 从 1 递增 |
| poLineNo | PO 行号 | integer | 继承 PO | 是 | 否 | 用于回写 PO 收货进度 |
| productCode | 商品编码 | string | 继承 PO | 是 | 否 | 商品快照，字母+数字 |
| productName | 商品名称 | string | 继承 PO | 是 | 否 | 商品快照 |
| productSpec | 规格 | string | 继承 PO | 否 | 否 | 无值展示 `-` |
| unit | 单位 | string | 继承 PO | 是 | 否 | 如 件、箱 |
| purchaseQty | 采购数量 | integer | PO 行 | 是 | 否 | 正整数，只读 |
| receivedQtyBefore | 已收数量 | integer | 系统累计 | 是 | 否 | 同一 PO 行历史已确认收货数量 |
| poUnreceivedQty | PO 未收数量 | integer | 系统计算 | 是 | 否 | `采购数量 - 已收数量` |
| actualReceiveQty | 本次实收数量 | integer | 仓管录入 | 是 | 是 | 正整数 `>0`，且 `≤ PO未收数量`；确认收货时强校验 |
| qualifiedQty | 合格数量 | integer | 质检填入 | 否 | 否 | 质检登记后填入；正整数 ≥0，且 ≤ 本次实收数量 |
| unqualifiedQty | 不合格数量 | integer | 质检填入 | 否 | 否 | = 本次实收数量 - 合格数量；系统自动计算 |
| unqualifiedReason | 不合格原因 | enum | 质检填入 | 条件必填 | 否 | 有不合格数量时必填；枚举+备注 |
| lineRemark | 行备注 | string | 仓管录入 | 否 | 是 | ≤200 字符 |

## 4. 关联展示字段

关联展示字段不参与收货单状态变更，用于列表和详情页追踪后续进度。

| 字段编码 | 字段名称 | 类型 | 来源 | 说明 |
|:--|:--|:--|:--|:--|
| putNo | 关联上架单号 | string | 上架环节 | 质检有合格品并生成上架单后，保存上架单号；未生成展示 `-` |

## 5. 枚举值

### 5.1 单据状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING_RECEIVE | 待收货 | PO 下推生成，未录入或已录入部分数量 |
| RECEIVING | 收货中 | 已录入部分实收数量，允许保存或继续录入，亦可确认收货 |
| PENDING_QC | 待质检 | 确认收货，数量锁定，库存记入冻结，等待进行质检登记 |
| PENDING_PUTAWAY | 待上架 | 质检登记完成且合格数量 > 0，已自动生成上架单 |
| EXCEPTION | 异常 | 质检登记合格数量 = 0，全部不合格，等待退货处理（二期） |
| COMPLETED | 已完成 | 关联的上架单全部完成上架，已全部完成分批过账转可用 |
| CANCELED | 已作废 | 待收货状态下作废，不回收单号 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| PO 未收数量 | `purchaseQty - receivedQtyBefore` | 按 PO 行实时计算 |
| 本单实收合计 | `Σ actualReceiveQty` | 仅统计本 RCV 明细行 |
| 打印次数 | `printCount + 1` | 每次打印成功后累加 |
| 不合格数量 | `actualReceiveQty - qualifiedQty` | 系统自动计算 |
| 本单合格合计 | `Σ qualifiedQty` | 仅统计本 RCV 明细行 |
| 本单不合格合计 | `Σ unqualifiedQty` | 仅统计本 RCV 明细行 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 实时失焦 | 数量格式、数量范围、备注长度 |
| 保存草稿 | 仓库/库区存在性、严重格式错误；允许暂未填满全部实收数量 |
| 确认收货 | 必填、正整数、`本次实收数量 ≤ PO未收数量`、状态为待收货/收货中 |
| 质检登记 | 必填（合格数量、不合格原因条件必填）、`合格数量 + 不合格数量 = 本次实收数量`、状态为待质检 |
