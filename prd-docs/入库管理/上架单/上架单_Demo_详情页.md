# 上架单_Demo_详情页

> 角色：Demo·详情页 | 类型：执行作业单
> 详情页用于查看上架单来源、货位明细、上架数量、库存流水和上下游关联。

## 1. 页面定位

上架单详情页面向研发、测试、产品复核，用于核对 PUT 是否由收货单 RCV 的质检合格品生成，实际上架货位和数量是否正确，库存是否在 PDA 上架确认时分批即时转为可用并即时生成对应的 FL。

## 2. 页面结构

```text
上架单详情
├─ 顶部摘要
│  ├─ 上架单号、状态、进度
│  └─ 返回列表、查看来源RCV、查看库存流水
├─ 来源信息
│  ├─ 来源收货单、仓库
│  └─ 创建时间、开始时间、完成时间
├─ 上架信息
│  ├─ 上架人、合格总数、已上架总数、待上架总数
│  └─ ERP回传状态、财务触发状态
├─ 上架明细
│  └─ 商品、合格数量、推荐货位、实际货位、本次/累计上架数量、实际上架确认时间
├─ 库存流水
│  └─ FL单号、时间、货位、商品、变动数量、变动后现存
└─ 操作记录
   └─ 生成、扫码、确认、分批过账、全部完成、回传
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态只展示 PUT 自身状态：待上架、上架中、已完成 |
| 来源信息 | 来源 RCV 只读，可跳转详情页 |
| 上架明细 | 数量右对齐，多货位分行展示 |
| 库存流水 | 展示各分批上架时即时过账产生的 FL 清单 |
| 操作记录 | 按时间倒序展示 |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 查看来源 RCV | 有 sourceRcvNo | 跳转收货单详情 |
| 查看库存流水 | 已生成对应 FL | 跳转库存流水明细 |
| 返回列表 | 全部状态 | 返回上架单列表 |

> 详情页不展示新增、审核、反审核、质检登记、退货处理操作。

## 6. 测试关注点

- PUT 详情能追溯来源 RCV。
- 状态不出现待审核、已审核等审核流状态。
- PDA 在上架中或已完成时已过账的记录不允许修改或回滚。
- 多货位上架时明细按实际货位分行展示。
- PDA 每次上架确认即时过账并生成对应的 FL 库存流水，不等待整单完成。
- 质检未过的 RCV 不应存在 PUT 详情。
