# 上架单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页为 PDA 上架作业页，重点描述扫码货位、数量确认和库存过账反馈；字段定义以 `上架单字段清单.md` 为准。

## 1. 页面定位

PDA 上架执行页用于仓管领取收货单质检登记合格品后自动生成的 PUT，依据推荐货位或现场人工指定货位完成上架。页面需要适合手套操作，大字体、大按钮，扫码优先。

## 2. 页面结构

```text
PDA 上架作业页
├─ 任务头
│  ├─ 上架单号、来源RCV、进度
│  └─ 仓库、状态、上架人
├─ 当前商品卡片
│  ├─ 商品名称、规格、单位
│  ├─ 合格数量、已上架数量、待上架数量
│  └─ 推荐货位
├─ 扫码区
│  └─ 扫实际货位条码
├─ 数量区
│  ├─ 本次上架数量
│  ├─ 加减步进
│  └─ 全部上架快捷按钮
├─ 反馈区
│  └─ 货位/数量/分批即时过账结果（显示 FL 流水号及入库可用成功提示）
└─ 底部大按钮
   └─ 确认本次上架并即时过账
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 领取任务 | 查看待上架任务，进入 PDA 作业页 |
| 查看推荐货位 | 推荐货位突出展示，可为空 |
| 扫实际货位 | 扫描后校验货位存在、启用、属于当前仓库 |
| 本次数量 | 默认可填待上架数量，支持步进调整 |
| 全部上架 | 将本次数量填为当前剩余待上架数量 |
| 确认本次上架并过账 | 校验通过后即时进行过账（扣减冻结库存转可用并生成该货位 FL），同时更新已录和待录数量；若累计完成则 PUT/RCV 联同转为已完成，触发回传及财务应付 |

## 4. 扫码与数量反馈

| 场景 | 反馈 |
|:--|:--|
| 扫货位正确 | 高亮货位，短提示音 |
| 货位不存在 | 红色提示、震动、语音提示“货位无效” |
| 货位停用 | 阻断，提示“货位已停用，不可上架” |
| 数量为 0 | 阻断，提示“本次上架数量必须大于 0” |
| 累计超量 | 阻断，提示“累计上架数量不能大于合格数量” |
| 本次确认过账 | 提示“过账成功，FLxxxxxx 已生成，库存转可用”并播放短语音 |
| 整单完成 | 自动返回任务列表，提示“此上架任务已全部上架完成” |

## 5. 页面状态

| PUT 状态 | 页面表现 |
|:--|:--|
| 待上架 | 展示当前商品、推荐货位，确认过账按钮可用 |
| 上架中 | 实时展示扣减后的剩余待上架数量，支持录入下一个货位，确认过账按钮可用 |
| 已完成 | 只读展示完成摘要、分批即时生成的流水 FL 清单，不可再编辑 |

## 7. 离线与同步

- PDA 支持离线缓存扫描和数量确认记录，网络恢复后自动同步。
- 离线期间仍需执行本地基础校验：数量不能超过待上架数量、货位条码不能为空。
- 同步时以后端最新状态为准；若 PUT 已完成或待上架数量变化，提示重新同步。
- 离线记录不得绕过累计数量上限。

## 8. 不展示内容

- 不展示新增上架单入口。
- 不展示审核按钮。
- 不展示质检登记、退货处理控件。
