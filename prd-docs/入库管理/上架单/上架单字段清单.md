# 上架单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为上架单字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- PUT 单号、来源 RCV、商品快照、合格数量、状态由系统生成或继承，不允许手工编辑。
- 数量字段按 `11-字段校验规范`：正整数、`>0`、`≤上限值`。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`。
- 空值展示 `-`，备注 ≤200 字符。
- 商品信息在单据中快照存储，主数据后续变更不影响历史上架记录。

## 2. 上架头字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| putNo | 上架单号 | string | 系统生成 | 是 | 否 | `PUT{YYYYMMDD}-{4位序号}`，如 `PUT20260705-0001` |
| sourceRcvNo | 来源收货单号 | string | 继承 RCV | 是 | 否 | 只读，用于追溯收货单 |
| warehouseCode | 仓库编码 | string | 继承 RCV | 是 | 否 | 必须为启用仓库 |
| warehouseName | 仓库名称 | string | 仓库档案 | 是 | 否 | 随仓库编码带出 |
| status | 上架状态 | enum | 系统 | 是 | 否 | `PENDING_PUTAWAY` 待上架、`PUTAWAYING` 上架中、`COMPLETED` 已完成 |
| putawayUserId | 上架人ID | string | PDA 当前用户 | 否 | 否 | 首次确认上架时写入 |
| putawayUserName | 上架人 | string | 用户档案 | 否 | 否 | 随上架人 ID 带出 |
| qualifiedTotalQty | 合格总数 | integer | 继承 收货单.合格数量 | 是 | 否 | 收货单中所有明细合格数量汇总，必须>0 |
| putawayTotalQty | 已上架总数 | integer | 系统累计 | 是 | 否 | `Σ actualPutawayQty` |
| pendingTotalQty | 待上架总数 | integer | 系统计算 | 是 | 否 | `合格总数 - 已上架总数` |
| progress | 上架进度 | string | 系统计算 | 是 | 否 | 如 `60/100` |
| remark | 备注 | string | 用户输入 | 否 | 是 | ≤200 字符 |

## 3. 上架明细字段

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|
| detailId | 明细ID | string | 系统生成 | 是 | 否 | 页面隐藏主键 |
| lineNo | 行号 | integer | 系统生成 | 是 | 否 | 从 1 递增 |
| productCode | 商品编码 | string | 继承 RCV | 是 | 否 | 商品快照，字母+数字 |

| unit | 单位 | string | 继承 RCV | 是 | 否 | 如 件、箱 |
| qualifiedQty | 合格数量 | integer | 继承 收货单.合格数量 | 是 | 否 | 正整数 `>0` |
| recommendedLocationCode | 推荐货位编码 | string | 系统计算 | 否 | 否 | 空闲货位优先，可为空 |
| recommendedLocationName | 推荐货位名称 | string | 货位档案 | 否 | 否 | 无推荐展示 `-` |
| actualLocationCode | 实际上架货位编码 | string | PDA 扫描/人工指定 | 是 | 否 | 必须存在、启用、属于当前仓库 |
| actualLocationBarcode | 实际货位条码 | string | PDA 扫描 | 是 | 否 | 扫描货位条码带入 |
| actualLocationName | 实际上架货位名称 | string | 货位档案 | 是 | 否 | 随货位编码带出 |
| currentPutawayQty | 本次上架数量 | integer | PDA 输入 | 是 | 是 | 正整数 `>0`，累计不得超过合格数量 |
| putawayQty | 累计上架数量 | integer | 系统累计 | 是 | 否 | 同一商品多货位累计 |

| lineStatus | 明细状态 | enum | 系统 | 是 | 否 | `PENDING` 待上架、`PUTAWAYING` 上架中、`COMPLETED` 已完成 |

## 5. 枚举值

### 5.1 上架状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING_PUTAWAY | 待上架 | 已生成，尚未确认上架 |
| PUTAWAYING | 上架中 | 已完成部分数量上架 |
| COMPLETED | 已完成 | 合格数量全部上架，已全部完成分批过账转可用 |

### 5.2 明细状态 `lineStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING | 待上架 | 当前明细尚未上架 |
| PUTAWAYING | 上架中 | 当前明细部分上架 |
| COMPLETED | 已完成 | 当前明细累计上架数量=合格数量 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 待上架数量 | `qualifiedQty - putawayQty` | 明细级剩余数量 |
| 已上架总数 | `Σ putawayQty` | 头部汇总 |
| 待上架总数 | `Σ remainingQty` | 头部汇总 |
| 本次上架上限 | `qualifiedQty - putawayQtyBefore` | PDA 确认时实时计算 |
| 完成条件 | `Σ putawayQty = Σ qualifiedQty` | 全部合格品完成上架 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 生成 PUT | 质检已完成、合格数量>0、来源 RCV 有效 |
| PDA 扫货位 | 货位存在、启用、属于当前仓库、可用于存储 |
| 数量确认 | 本次上架数量为正整数，累计上架数量≤合格数量 |
| 完成上架 | 所有明细累计上架数量=合格数量 |
