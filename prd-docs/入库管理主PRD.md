# WMS 入库管理主PRD

> 版本 V1.0 | 2026-07-06 | 强盛科技 WMS

## 1. 业务背景

仓管员收到供应商到货 → 清点实物 → 质检 → 上架到货位。现有痛点：手工 Excel 记录，收货到上架平均 45 分钟，漏记错记频发。

## 2. 功能范围

**In Scope**：PC 创建收货单（关联进销存 PO）→ 质检判定 → PDA 上架扫描 → 库存可用
**Out Scope**：无单收货、供应商预约到货、上架路径优化（二期）

## 3. 单据体系

| 单据 | 前缀 | 说明 |
|:--|:--|:--|
| 收货单 RCV | RCV{日期}-{序号} | 关联采购订单 PO |
| 上架单 PUT | PUT{日期}-{序号} | PDA 扫描货位确认 |

## 4. 状态机

| 状态 | 含义 | 可用操作 |
|:--|:--|:--|
| 草稿 DRAFT | 已创建，待清点 | 编辑/删除/作废/确认收货 |
| 已收货 RECEIVED | 清点完成，待质检 | 质检通过/质检不通过 |
| 已上架 PUTAWAY | 上架完成，库存可用 | —（终态） |
| 已作废 VOIDED | 失效 | —（终态） |

流转：草稿→已收货(确认收货)→已上架(PDA上架确认)；草稿→已作废

## 5. 核心业务规则

- R01：收货单必须关联已审核的采购订单 PO，供应商/仓库/商品明细继承 PO
- R02：实收数量 ≤ PO 未收货数量，超收阻断
- R03：质检不合格商品标记退货，合格品进入上架流程
- R04：PDA 扫描货位确认上架后，库存状态从冻结→可用，回写进销存库存
- R05：三数量口径——采购数量(PO) / 实收数量(RCV) / 上架数量(PUT)

## 6. 验收重点

| 验收项 | 预期 |
|:--|:--|
| 超收阻断 | 实收>PO未收货时标红阻断 |
| 质检状态 | 质检期间库存=冻结，不可销售 |
| 上架完成 | PDA确认后库存可用+N，生成 FL |
