# WMS 库存管理主PRD

> 版本 V1.0 | 2026-07-06 | 强盛科技 WMS

## 1. 业务背景

全系统库存统一视图——按仓库/库区/货位/商品/批次/状态多维度实时查询。解决"不知道有多少可卖、多少被锁、多少在质检"的问题。

## 2. 功能范围

**In Scope**：
- 即时库存查询（现存/占用/冻结/在途四状态同屏）
- 库存流水 FL（所有变动自动记录，只读追溯）
- 库存预警（低于安全库存标黄，低于最低标红）
- 零库存过滤开关

**Out Scope**：自动补货建议、库存成本核算、库龄分析（二期）

## 3. 单据定位

| 项目 | 内容 |
|:--|:--|
| 层级 | 只读查询台账，无状态机 |
| 核心职责 | 提供全系统库存的 SSOT 数据源 |
| 数据来源 | 系统基于期初 + 所有 FL 实时聚合 |
| 库存流水 FL | FL{日期}-{8位序号}，由入库/出库/盘点/调拨/报损确认时自动生成 |

## 4. 库存状态（四口径）

| 状态 | 含义 | 触发 |
|:--|:--|:--|
| 可用 | 可销售可出库 | 上架完成、解冻 |
| 占用 | 销售订单已锁定 | SO 审核 |
| 冻结 | 质检中/盘点中 | QC/盘点触发 |
| 在途 | 调拨途中 | 调出已确认→调入未确认 |

公式：可用 = 现存 - 占用 - 冻结

## 5. 核心业务规则

- R01：库存变动仅发生在执行层确认时（入库确认/出库交运/盘点调整），禁止手动修改
- R02：每次变动自动生成 FL——时间/仓库/货位/商品/类型/变动量(±)/变动后现存
- R03：变动方向——入库+/出库-/盘盈+/盘亏-
- R04：安全库存从商品档案读取，可用<安全时预警标黄 Tag
- R05：现存<0 时红色展示（库存倒挂）
- R06：开启"不显示零库存"开关 → 过滤现存≤0的行
- R07：FL 默认按发生时间降序，日期筛选跨度 ≤365 天

## 6. 验收重点

| 验收项 | 预期 |
|:--|:--|
| 四状态同屏 | 可用/占用/冻结/在途同表展示 |
| 库存预警 | 可用<安全→黄Tag"低于安全库存" |
| 零库存过滤 | 开关开启后零库存行消失 |
| FL 只读 | 无新增/编辑/删除按钮 |
| 日期超限 | 跨度>365天阻断提示 |
