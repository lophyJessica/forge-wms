# 报损单_用例数据推演

> 角色：用例数据推演 | 类型：业务单据类
> 使用 2026 年示例数据，覆盖手动报损和调拨损耗两类来源，并写清现存、占用、冻结、可用三口径走账。

## 1. 基础数据

| 类型 | 编码 | 名称 | 说明 |
|:--|:--|:--|:--|
| 仓库 | WH-SH-01 | 上海一仓 | 启用 |
| 仓库 | WH-CD-01 | 成都仓 | 启用 |
| 货位 | LOC-A0102 | A-01-02 | 存储区货位 |
| 货位 | LOC-DIFF-CD | 调拨差异待核销 | Demo 展示口径，正式货位待确认 |
| 商品 | SKU004 | 得力多功能计算器 | 单位：台 |
| 商品 | SKU002 | 晨光按动式中性笔黑色 | 单位：支 |
| 审核人 | U-MGR-001 | 仓库主管-李娜 | 有报损审核权限 |
| 财务 | U-FIN-002 | 财务-周敏 | 有资产核销复核权限 |

## 2. 公式口径

| 项 | 公式/规则 |
|:--|:--|
| 可用 | `现存 - 占用 - 冻结` |
| 草稿 | 不冻结、不扣现存 |
| 待审核 | 冻结 `+N`，现存不变 |
| 已审核 | 现存 `-N`，本单冻结 `-N`，生成来源 BL 的 FL |
| 已驳回 | 本单冻结释放，现存不变 |

## 3. 用例 1：手动报损，损坏核销

### 3.1 输入

| 字段 | 值 |
|:--|:--|
| 报损单号 | BL20260706-0001 |
| 来源 | `manual` |
| 仓库 | WH-SH-01 上海一仓 |
| 货位 | LOC-A0102 A-01-02 |
| 商品 | SKU004 得力多功能计算器 |
| 报损类型 | `goods_damage` 货物损坏 |
| 报损原因 | `damaged` 损坏 |
| 报损数量 N | 6 |
| 申请人 | 仓管员-王强 |
| 审核人 | 仓库主管-李娜 |

### 3.2 库存快照

| 时点 | BL 状态 | 现存 | 占用 | 冻结 | 可用 | 变化说明 |
|:--|:--|--:|--:|--:|--:|:--|
| 初始 | - | 120 | 20 | 0 | 100 | `120-20-0=100` |
| 保存草稿 | 草稿 | 120 | 20 | 0 | 100 | 库存不变 |
| 提交审核 | 待审核 | 120 | 20 | 6 | 94 | 冻结+6，可用减少，现存不变 |
| 审核核销 | 已审核 | 114 | 20 | 0 | 94 | 现存-6，本单冻结释放 |

### 3.3 FL 结果

| 字段 | 值 |
|:--|:--|
| flow_no | FL20260706-00000041 |
| occurred_at | 2026-07-06 10:30:18 |
| source_order_type | `BL` |
| source_order_no | BL20260706-0001 |
| change_type | `STOCK_LOSS` |
| change_qty | -6 |
| qty_on_hand_after | 114 |

### 3.4 校验点

| 校验 | 结果 |
|:--|:--|
| 草稿不动库存 | 现存、占用、冻结、可用均不变 |
| 待审核不扣现存 | 现存仍为 120 |
| 待审核冻结 | 冻结从 0 到 6，可用从 100 到 94 |
| 核销后现存 | `120 - 6 = 114` |
| 核销后可用 | `114 - 20 - 0 = 94` |

## 4. 用例 2：调拨损耗，自动生成 BL

### 4.1 输入

| 字段 | 值 |
|:--|:--|
| 调拨单号 | TR20260706-0002 |
| 报损单号 | BL20260706-0002 |
| 来源 | `transfer_diff` |
| 调出仓 | WH-SZ-01 深圳仓 |
| 调入仓 | WH-CD-01 成都仓 |
| 商品 | SKU002 晨光按动式中性笔黑色 |
| 调出数量 | 80 |
| 实收数量 | 76 |
| 报损数量 N | 4 |
| 报损原因 | `transfer_loss` 调拨损耗 |

### 4.2 TR 差异快照

| 时点 | 调出数量 | 实收数量 | 差异数量 | TR 处理 |
|:--|--:|--:|--:|:--|
| 调出确认 | 80 | - | - | 调出仓现存-80，调入仓在途+80 |
| 调入清点 | 80 | 76 | 4 | 调入仓按 76 入库 |
| 差异确认 | 80 | 76 | 4 | 系统生成 BL20260706-0002 |

### 4.3 BL 三口径走账

| 时点 | BL 状态 | 现存 | 占用 | 冻结 | 可用 | 变化说明 |
|:--|:--|--:|--:|--:|--:|:--|
| BL 生成前 | - | 4 | 0 | 0 | 4 | 差异待核销数量，Demo 展示口径 |
| 自动生成后 | 待审核 | 4 | 0 | 4 | 0 | 调拨损耗待审核，冻结+4 |
| 审核核销后 | 已审核 | 0 | 0 | 0 | 0 | 现存-4，冻结-4 |

### 4.4 FL 结果

| 字段 | 值 |
|:--|:--|
| flow_no | FL20260706-00000042 |
| occurred_at | 2026-07-06 15:18:09 |
| source_order_type | `BL` |
| source_order_no | BL20260706-0002 |
| source_line_id | TR20260706-0002-L2 |
| change_type | `STOCK_LOSS` |
| change_qty | -4 |
| qty_on_hand_after | 0 |

### 4.5 校验点

| 校验 | 结果 |
|:--|:--|
| BL 数量 | `80 - 76 = 4` |
| BL 原因 | 固定为 `transfer_loss` 调拨损耗 |
| 关联 TR | `related_order_no=TR20260706-0002` |
| 待审核不核销 | 待审核阶段现存仍为 4 |
| 核销后现存 | `4 - 4 = 0` |
| 幂等 | 同一 TR 行不得生成第二张有效 BL 或重复 FL |

> 不确定性：调拨差异在正式库存台账中是冻结现存、在途待核销还是差异责任库存，context 未明确。本用例按二期 Demo 的 BL 资产核销展示口径写三口径，落地时需与 TR 在途清零规则统一。

## 5. 用例 3：审核驳回，释放冻结

### 5.1 输入

| 字段 | 值 |
|:--|:--|
| 报损单号 | BL20260706-0003 |
| 来源 | `manual` |
| 商品 | SKU004 得力多功能计算器 |
| 报损数量 | 5 |
| 初始库存 | 现存 60，占用 10，冻结 0，可用 50 |

### 5.2 库存快照

| 时点 | BL 状态 | 现存 | 占用 | 冻结 | 可用 | 说明 |
|:--|:--|--:|--:|--:|--:|:--|
| 提交审核前 | 草稿 | 60 | 10 | 0 | 50 | 草稿不影响库存 |
| 提交审核后 | 待审核 | 60 | 10 | 5 | 45 | 冻结+5 |
| 审核驳回后 | 已驳回 | 60 | 10 | 0 | 50 | 释放冻结，现存不变 |

### 5.3 期望结果

- 审核人必须填写驳回原因。
- 已驳回 BL 不生成核销 FL。
- 若库存流水口径要求记录解冻，可生成 `UNFREEZE`，但不生成来源 BL 的 `STOCK_LOSS` 核销 FL。

## 6. 用例 4：数量校验失败

| 输入 | 系统判断 | 期望结果 |
|:--|:--|:--|
| 报损数量为空 | 必填不满足 | 阻断保存/提交 |
| 报损数量=0 | 正整数不满足 | 阻断保存/提交 |
| 可用 15，报损 20 | 可冻结数量不足 | 阻断提交，保持草稿 |
| 原因=其它，说明为空 | 条件必填不满足 | 阻断提交 |

## 7. 用例 5：重复核销幂等

| 输入 | 系统判断 | 期望结果 |
|:--|:--|:--|
| 已审核 BL 再次点击核销 | 状态已为 `approved` | 阻断，不新增 FL |
| 网络重试同一核销请求 | 幂等键已存在 | 返回已处理结果 |
| 同一 TR 行重复生成 BL | 来源行已有有效 BL | 不再生成第二张有效 BL |

## 8. 走查清单

| 检查项 | 预期 |
|:--|:--|
| BL 前缀 | 符合 `BL{YYYYMMDD}-{4位序号}` |
| 状态 | 草稿、待审核、已审核、已驳回 |
| 来源 | 手动、调拨差异 |
| 原因 | 手动含损坏/过期/其它；调拨差异固定调拨损耗 |
| 草稿 | 不冻结、不扣现存 |
| 待审核 | 冻结报损数量，现存不变 |
| 已审核 | 现存-N，生成来源 BL 的 FL |
| 已驳回 | 释放冻结，不生成核销 FL |
| 三口径 | 可用始终按 `现存 - 占用 - 冻结` 计算 |
