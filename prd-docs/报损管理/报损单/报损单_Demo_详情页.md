# 报损单_Demo_详情页

> 角色：Demo·详情页 | 类型：业务单据类
> 详情页用于查看报损单来源、明细、冻结状态、审核结论、核销结果、关联 TR 和关联 FL。

## 1. 页面定位

报损单详情页向研发、测试、产品复核提供完整核对视图。它需要证明 BL 覆盖两类来源、一道审核、资产核销和库存流水追溯，并明确草稿/待审核不扣现存。

## 2. 页面结构

```text
报损单详情
├─ 顶部摘要
│  ├─ 报损单号、状态、来源、报损原因
│  └─ 返回列表、编辑、提交审核、审核通过并确认核销、审核驳回、查看FL
├─ 报损头信息
│  ├─ 仓库、调出仓/调入仓、关联单号、报损原因、原因说明
│  └─ 申请人、审核人、创建时间、提交时间、审核时间、核销时间
├─ 报损明细
│  ├─ 货位、商品、批次、单位、报损数量
│  └─ 核销前现存、占用、冻结、可用、核销后现存
├─ 库存影响
│  ├─ 草稿/待审核/已审核/已驳回口径说明
│  └─ 三口径公式复核
├─ 审核信息
│  ├─ 审核结论、驳回原因
│  └─ 审核人、审核时间
├─ 关联单据
│  ├─ 来源 TR
│  └─ 关联 FL
└─ 操作记录
   └─ 创建、提交、冻结、审核、核销、驳回
```

## 3. 展示规则

| 区域 | 展示规则 |
|:--|:--|
| 顶部摘要 | 状态只展示 BL 自身状态：草稿、待审核、已审核、已驳回 |
| 来源信息 | 调拨差异展示关联 TR；手动来源关联单号展示 `-` |
| 报损明细 | 数量右对齐；现存/占用/冻结/可用展示提交审核时快照 |
| 库存影响 | 按来源展示核销口径：手动报损审核核销才现存-N；调拨损耗审核核销扣差异待核销、不动现存 |
| 审核信息 | 待审核展示审核动作；已驳回展示驳回原因 |
| 关联 FL | 已审核后展示；草稿、待审核、已驳回展示 `-` |
| 操作记录 | 按时间升序展示状态动作，不允许编辑 |

## 4. 动作规则

| 动作 | 展示条件 | 说明 |
|:--|:--|:--|
| 编辑 | 状态=草稿且来源=手动 | 进入新增编辑页 |
| 提交审核 | 状态=草稿 | 校验通过后进入待审核并冻结 |
| 审核通过并确认核销 | 状态=待审核 | 手动报损现存-N；调拨损耗差异待核销-N；生成 FL，状态进入已审核 |
| 审核驳回 | 状态=待审核 | 填写驳回原因，释放冻结，状态进入已驳回 |
| 查看 TR | 来源=调拨差异且有关联 TR | 跳转调拨单详情 |
| 查看 FL | 状态=已审核且有关联 FL | 跳转库存流水 |
| 返回列表 | 全部状态 | 返回报损单列表 |

> 详情页不展示删除、反审核、二次审批、独立核销单等入口。

## 9. 测试关注点

- 详情页能完整展示状态链路和操作记录。
- 待审核状态必须能看到“审核通过并确认核销”和“审核驳回”动作。
- 已审核状态能追溯 FL，FL 来源单号为当前 BL。
- 已驳回状态不生成核销 FL，现存不变。
- 页面明确展示可用公式和分来源核销口径：手动报损确认核销才现存-N，调拨损耗确认核销扣差异待核销且现存不变。
