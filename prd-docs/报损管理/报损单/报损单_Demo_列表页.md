# 报损单_Demo_列表页

> 角色：Demo·列表页 | 类型：业务单据类
> 页面只描述结构、控件、交互和 Mock；字段定义以 `报损单字段清单.md` 为准，不重复维护字段枚举。

## 1. 页面定位

报损单列表页用于 PC 端创建和跟踪 BL，覆盖草稿、待审核、已审核、已驳回状态。列表页承担查询、创建入口、审核入口和 FL 追溯入口，不承载库存字段枚举定义。

## 2. 页面入口

| 入口 | 说明 |
|:--|:--|
| 菜单 | 报损管理 / 报损单 |
| 关联跳转 | 调拨单详情、库存查询、库存流水、报损单详情 |

## 3. 页面结构

```text
报损单列表
├─ 顶部操作区
│  └─ 新建报损单
├─ 筛选区
│  ├─ 关键词：报损单号/关联单号/货位/商品/申请人/审核人
│  ├─ 状态、来源、报损原因、仓库
│  ├─ 创建时间范围、核销时间范围
│  └─ 查询、重置
├─ 表格区
│  ├─ 报损单号、状态、来源、关联单号、仓库
│  ├─ 报损原因、报损总数、明细行数
│  ├─ 申请人、审核人、创建时间、提交时间、核销时间
│  └─ 行操作：查看详情、编辑、提交审核、审核、查看FL
└─ 分页
   └─ 默认 20 条/页
```

## 4. 行操作规则

| BL 状态 | 行操作 |
|:--|:--|
| 草稿 | 查看详情、编辑、提交审核 |
| 待审核 | 查看详情、审核 |
| 已审核 | 查看详情、查看 FL |
| 已驳回 | 查看详情 |

> 按钮不可用时隐藏。列表页不展示删除入口，不展示反审核入口，不展示多级审批入口。

## 5. 查询规则

- 关键词支持报损单号、关联单号、货位编码、商品编码、申请人、审核人模糊查询。
- 状态筛选仅使用 BL 自身状态：草稿、待审核、已审核、已驳回。
- 来源筛选支持手动、调拨差异。
- 默认按创建时间倒序展示。
- PC 表格默认 20 条/页。
- 空值统一展示 `-`。

## 6. 新建报损单入口

| 区域 | 字段/控件 | 说明 |
|:--|:--|:--|
| 基础信息 | 来源、仓库、报损原因 | 手动新建时来源固定为手动 |
| 明细信息 | 货位、商品、报损数量 | 数量为正整数，提交审核时校验可冻结数量 |
| 人员信息 | 申请人、审核人 | 审核人必须具备报损审核权限 |
| 备注信息 | 原因说明、备注 | 其它原因时原因说明必填 |
| 动作 | 保存草稿、提交审核 | 提交审核后进入待审核并冻结 |

## 8. 空状态与异常态

| 状态 | 展示 |
|:--|:--|
| 无数据 | `暂无报损单` |
| 查询无结果 | `未找到符合条件的报损单` |
| 列表加载失败 | 保留筛选条件，提示 `报损单列表加载失败，请重试` |
| 提交审核失败 | 展示失败原因，如 `可报损库存不足` |
| 审核核销失败 | 展示失败原因，如 `库存核销失败，请重试` |

## 9. 测试关注点

- 列表页可新建手动 BL，但不能删除 BL。
- 草稿可编辑和提交审核，提交后进入待审核。
- 待审核才显示审核入口。
- 已审核才显示查看 FL。
- 调拨差异来源的 BL 显示关联 TR，不提供编辑来源原因入口。
