# 报损单_Demo_新增编辑页

> 角色：Demo·新增编辑页 | 类型：业务单据类
> 本页覆盖手动报损的新增、草稿编辑和提交审核；字段定义以 `报损单字段清单.md` 为准。

## 1. 页面定位

新增编辑页用于仓管员或仓库主管创建手动来源 BL。调拨差异来源的 BL 由系统自动生成，不从本页新建；进入本页查看时应只读展示或跳转详情。

## 2. 页面结构

```text
报损单新增编辑
├─ 顶部摘要
│  ├─ 标题：新建报损单 / 编辑报损单
│  └─ 返回列表、保存草稿、提交审核
├─ 基础信息
│  ├─ 报损单号、来源、状态、仓库
│  ├─ 报损原因、原因说明
│  └─ 申请人、审核人、备注
├─ 报损明细
│  ├─ 添加明细、移除明细
│  ├─ 货位、商品、批次、单位
│  ├─ 当前现存、当前占用、当前冻结、当前可用
│  └─ 报损数量、行备注
└─ 底部操作
   ├─ 保存草稿
   └─ 提交审核
```

## 3. 控件与交互

| 区域 | 控件 | 交互说明 |
|:--|:--|:--|
| 来源 | Readonly | 手动新增固定展示“手动” |
| 仓库 | SelectWithSearch | 选择启用仓库，按用户权限过滤 |
| 报损原因 | Select | 原因=其它时展示并必填原因说明 |
| 申请人 | SelectWithSearch | 默认当前用户，可按权限调整 |
| 审核人 | SelectWithSearch | 仅展示有审核权限账号 |
| 添加明细 | Button | 新增一行空明细 |
| 货位 | SelectWithSearch/ScanInput | 选择后带出货位当前库存 |
| 商品 | SelectWithSearch/ScanInput | 按货位库存过滤可选商品 |
| 报损数量 | NumberInput | 正整数，提交时校验不超过可报损数量 |
| 保存草稿 | Button | 宽松校验，不冻结库存 |
| 提交审核 | Button | 全量校验，成功后进入待审核并冻结 |

## 4. 页面状态

| BL 状态 | 页面表现 |
|:--|:--|
| 新建 | 可编辑，未生成或保存后生成 BL 单号 |
| 草稿 | 可编辑，可保存草稿或提交审核 |
| 待审核 | 不允许编辑，提示“报损单待审核” |
| 已审核 | 只读，展示核销结果和 FL |
| 已驳回 | 只读，展示驳回原因 |

## 5. 校验规则

| 时机 | 校验 |
|:--|:--|
| 失焦 | 数量为正整数、备注长度、原因说明长度 |
| 保存草稿 | 仓库、报损原因、至少一条明细 |
| 提交审核 | 所有必填、审核人、可冻结数量、其它原因说明、明细数量上限 |
| 明细添加 | 同一货位+商品+批次重复时提示合并 |
| 明细移除 | 草稿可移除；提交后不允许移除 |

## 7. 提交审核反馈

| 场景 | 反馈 |
|:--|:--|
| 提交成功 | Toast：`已提交审核`，跳转详情页 |
| 可报损库存不足 | 阻断提交，定位到明细行 |
| 审核人无权限 | 阻断提交，提示重新选择审核人 |
| 原因说明缺失 | 阻断提交，原因说明输入框标红 |
| 冻结失败 | 保持草稿或提交前状态，提示失败原因 |

## 8. 不展示内容

- 不展示删除按钮。
- 不展示审核通过、驳回、确认核销按钮。
- 不展示调拨差异来源字段的编辑入口。
- 不展示多级审批配置。

## 9. 测试关注点

- 保存草稿不影响库存。
- 提交审核才触发报损冻结，现存不变。
- 报损数量必须为正整数。
- 原因=其它时必须填写原因说明。
- 调拨差异 BL 不应从本页人工创建。
