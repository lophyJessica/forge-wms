# 盘点单_Demo_执行页

> 角色：Demo·执行页 | 类型：执行作业单
> 本页为 PDA 扫描实盘作业页，重点描述扫货位、扫商品、录实盘数和提交明细；字段定义以 `盘点单字段清单.md` 为准。

## 1. 页面定位

PDA 盘点执行页用于盘点人在 CK 进入盘点中后执行现场实盘。页面遵守 PDA 端规范：大字体、大按钮，扫码优先于手动输入，关键操作提供语音和震动反馈。

## 2. 页面结构

```text
PDA 盘点作业页
├─ 任务头
│  ├─ 盘点单号、盘点类型、状态、进度
│  └─ 仓库、盘点范围、盘点人
├─ 当前明细卡片
│  ├─ 货位、商品、规格、单位
│  ├─ 系统数（仅明盘展示）
│  └─ 实盘数、差异提示（提交后展示）
├─ 扫码区
│  ├─ 扫货位
│  └─ 扫商品
├─ 数量区
│  ├─ 实盘数输入
│  ├─ 加减步进
│  └─ 清零/确认数量
├─ 明细列表
│  └─ 待盘、已盘、待重盘分组
└─ 底部大按钮
   ├─ 确认本行
   └─ 完成实盘
```

## 3. 核心交互

| 交互 | 说明 |
|:--|:--|
| 加载任务 | 仅展示状态为盘点中的 CK |
| 扫货位 | 校验货位是否在盘点范围内；动盘场景锁定当前扫描货位 |
| 扫商品 | 校验商品是否匹配当前明细或盘点范围 |
| 明盘展示 | `count_type=OPEN` 时展示系统数 |
| 盲盘展示 | `count_type=BLIND` 时隐藏系统数，仅展示商品和实盘输入 |
| 实盘数输入 | 支持扫码后手动录入，必须为整数且 `>=0` |
| 确认本行 | 写入实盘数、实盘人、实盘时间，明细状态变为已盘 |
| 完成实盘 | 全部明细已盘后展示，提交后由系统生成差异报告 |

## 4. 扫码反馈

| 场景 | 反馈 |
|:--|:--|
| 扫货位正确 | 高亮货位，短提示音 |
| 货位不在范围 | 红色提示、震动、语音提示“货位不在本次盘点范围内” |
| 扫商品正确 | 高亮商品，允许输入实盘数 |
| 商品不匹配 | 红色提示、震动、语音提示“商品不匹配” |
| 实盘数为空 | 阻断提交，提示“请输入实盘数” |
| 实盘数为负数 | 阻断提交，提示“实盘数不能小于 0” |
| 离线缓存成功 | 顶部展示离线标识，提示“已离线保存，联网后同步” |

## 5. 页面状态

| CK 状态 | 页面表现 |
|:--|:--|
| 草稿 | 不允许进入 PDA 作业，提示“盘点单未开始” |
| 盘点中 | 展示扫码和实盘控件 |
| 待审核 | 只读展示已提交结果，不允许继续扫码 |
| 已调整 | 只读展示结果和关联 FL，不允许继续扫码 |

## 7. 离线与同步

- PDA 支持离线缓存实盘记录，网络恢复后自动同步。
- 离线期间只能缓存本机已拉取的 CK 明细，不允许新增盘点范围外的货位。
- 同步时以后端最新 CK 状态为准；若 CK 已待审核或已调整，离线记录不得覆盖。
- 同步冲突需保留本地记录并提示盘点人联系主管处理。

## 8. 不展示内容

- 不展示主管审核按钮。
- 不展示确认调整按钮。
- 不展示库存流水生成按钮。
- 不展示独立差异调整单入口。

## 9. 测试关注点

- 明盘显示系统数，盲盘隐藏系统数。
- PDA 必须先扫货位再扫商品。
- 实盘数允许 0，但不允许负数和小数。
- 全部明细已盘后才允许完成实盘。
- PDA 完成实盘后不直接更新现存，不直接生成 FL；库存调整由待审核阶段主管确认触发。
