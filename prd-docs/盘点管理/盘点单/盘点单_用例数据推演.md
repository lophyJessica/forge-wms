## 3. 用例 1：创建并开始盘点

### 输入

| 字段 | 值 |
|:--|:--|
| count_no | CK20260706-0001 |
| count_type | `BLIND` |
| scope_type | `LOCATION` |
| location_codes | LOC-A0102、LOC-A0105 |
| counter_name | 盘点员-陈晨 |
| reviewer_name | 仓库主管-李娜 |

### 状态快照

| 时点 | CK 状态 | 货位冻结 | 系统数快照 | 可执行动作 |
|:--|:--|:--|:--|:--|
| 保存草稿后 | 草稿 | 否 | 未锁定 | 编辑范围、开始盘点 |
| 开始盘点后 | 盘点中 | 是 | 已写入明细 | PDA 实盘、生成差异报告 |

### 期望结果

- 点击开始盘点后，A-01-02、A-01-05 进入冻结态。
- 命中这两个货位的入库、出库、调拨请求被阻断。
- PDA 因 `count_type=BLIND` 不展示系统数。

## 4. 用例 2：盘盈走账，现存增加

### 输入

| 字段 | 值 |
|:--|:--|
| 货位 | A-01-02 |
| SKU | SKU004 |
| 系统数 | 100 |
| 实盘数 | 104 |
| 差异数 | +4 |
| 差异率 | 4% |

### 库存快照

| 时点 | 现存 | 占用 | 冻结 | 可用 | 说明 |
|:--|--:|--:|--:|--:|:--|
| 锁定前 | 100 | 10 | 0 | 90 | `可用=100-10-0` |
| 盘点中锁定后 | 100 | 10 | 90 | 0 | 可用转冻结，现存和占用不变 |
| 已调整解冻后 | 104 | 10 | 0 | 94 | 盘盈现存 +4，占用不变，可用按公式重算 |

### FL 结果

| 字段 | 值 |
|:--|:--|
| flow_no | FL20260706-00000031 |
| source_order_no | CK20260706-0001 |
| change_type | `STOCK_GAIN` |
| change_qty | +4 |
| qty_on_hand_after | 104 |

### 期望结果

- 审核通过并确认调整后，SKU004 现存从 100 增加到 104。
- 不修改占用；可用不是单独手工调整，而是按公式从 90 变为 94。
- 生成一条来源为 CK 的盘盈 FL。

## 6. 用例 4：差异率 >5% 触发主管重盘

### 输入

| 字段 | 值 |
|:--|:--|
| 货位 | B-02-01 |
| SKU | SKU007 |
| 系统数 | 80 |
| 首次实盘数 | 72 |
| 差异数 | -8 |
| 差异率 | 10% |

### 状态快照

| 时点 | CK 状态 | 行状态 | 重盘次数 | 期望结果 |
|:--|:--|:--|--:|:--|
| 差异报告生成后 | 待审核 | 已盘 | 0 | 展示差异率 10% |
| 主管处理后 | 盘点中 | 待重盘 | 1 | 主管重盘，保持冻结 |

### 期望结果

- 由于 10% > 5%，不允许直接“审核通过并确认调整”。
- 主管点击“要求重盘”后，CK 从待审核回到盘点中。
- 重盘明细必须重新 PDA 扫描并提交实盘数。

## 7. 用例 5：生成差异报告

| 条件 | 判断 | 期望结果 |
|:--|:--|:--|
| 全部明细已盘 | 满足 | 允许生成差异报告，状态变为待审核 |
| 存在待盘明细 | 不满足 | 隐藏生成差异报告按钮 |
| 差异行数 >0 | 满足 | 报告展示差异数、差异率和差异类型 |
| 无差异 | 满足 | 报告展示无差异，审核通过后不生成盘盈/盘亏 FL |

## 8. 用例 6：盘点中冻结阻断出入库

### 输入

- CK20260706-0001 状态为盘点中。
- 货位 A-01-02 已冻结。
- 上架任务尝试向 A-01-02 上架 5 台 SKU004。

### 期望结果

| 判断 | 结果 |
|:--|:--|
| 目标货位处于盘点冻结 | 阻断上架确认 |
| 库存影响 | 现存、占用、冻结、可用均不因该上架请求变化 |
| 提示 | `货位盘点中，禁止出入库` |

## 9. 用例 7：幂等生成 FL

| 输入 | 系统判断 | 期望结果 |
|:--|:--|:--|
| 已调整 CK 再次点击确认调整 | 状态已为 `adjusted` | 阻断重复调整，不新增 FL |
| 网络重试同一确认调整请求 | 来源 CK 明细已有有效 FL | 返回已处理结果，不重复增加或减少现存 |

## 10. 走查清单

| 检查项 | 预期 |
|:--|:--|
| CK 前缀 | 符合 `CK{YYYYMMDD}-{4位序号}` |
| 状态 | 草稿、盘点中、待审核、已调整 |
| 冻结 | 盘点中货位冻结，禁止出入库 |
| 差异报告 | 全部明细已盘后生成 |
| 主管审核 | 差异报告必须经主管处理 |
| 超阈值 | 差异率 >5% 回到盘点中重盘 |
| 调整 | 确认调整才更新现存并生成 FL |
| 库存口径 | 盘盈现存+，盘亏现存-，占用不变，可用按公式重算 |
