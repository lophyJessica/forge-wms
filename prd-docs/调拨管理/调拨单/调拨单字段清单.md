# 调拨单字段清单

> 角色：字段清单(SSOT) | 类型：执行作业单
> 本文件为调拨单字段唯一事实来源；主 PRD、业务规则、流程推演和 Demo 页面引用本文件，不重复维护字段口径。

## 1. 字段设计原则

- TR 单号由系统按 `TR{YYYYMMDD}-{4位序号}` 生成，不可编辑。
- 来源为 ERP 下发时，ERP来源单号、调拨仓库、调拨明细由 ERP 带入，WMS 只执行，不改来源口径。
- 数量字段遵循 `11-字段校验规范`：整数，调出数量 `>0`，实收数量 `≥0` 且不得大于调出数量。
- 商品信息在单据明细中快照存储，避免主数据后续变更影响历史单据。
- 状态字段只读，必须通过动作按钮触发变更。
- 日期格式 `YYYY-MM-DD`，时间格式 `YYYY-MM-DD HH:mm:ss`，空值展示 `-`。
- 备注、差异说明不超过 200 字符。

## 2. 调拨头字段（调出仓/调入仓/类型/来源）

|:--|:--|:--|:--|:--:|:--:|:--|
| transferNo | 调拨单号 | string | 系统生成 | 是 | 否 | `TR{YYYYMMDD}-{4位序号}`，如 `TR20260705-0001` |
| transferType | 调拨类型 | enum | 用户选择/ERP 下发 | 是 | 草稿可编辑 | `WAREHOUSE_TRANSFER` 仓间调拨、`ERP_TRANSFER` ERP 下发调拨 |
| sourceType | 来源 | enum | 系统/接口 | 是 | 否 | `WMS_MANUAL` WMS 手动、`ERP_ISSUED` ERP 下发 |
| sourceErpOrderNo | ERP来源单号 | string | ERP | 条件必填 | 否 | ERP 下发时必填并只读；WMS 手动为空展示 `-` |
| outWarehouseCode | 调出仓编码 | string | 用户选择/ERP 下发 | 是 | 草稿可编辑 | 必须为启用仓库 |
| outWarehouseName | 调出仓名称 | string | 仓库档案 | 是 | 否 | 随调出仓编码带出 |
| inWarehouseCode | 调入仓编码 | string | 用户选择/ERP 下发 | 是 | 草稿可编辑 | 必须为启用仓库，不能等于调出仓 |
| inWarehouseName | 调入仓名称 | string | 仓库档案 | 是 | 否 | 随调入仓编码带出 |
| status | 调拨状态 | enum | 系统 | 是 | 否 | `DRAFT` 草稿、`TRANSFERRED_OUT` 已调出（在途）、`TRANSFERRED_IN` 已调入（完成） |
| transferOutQty | 调出总数 | integer | 系统汇总 | 是 | 否 | `Σ lineTransferOutQty` |
| actualReceiveQty | 实收总数 | integer | 系统汇总 | 是 | 否 | `Σ lineActualReceiveQty`，调入确认前可为 0 |
| diffQty | 差异总数 | integer | 系统计算 | 是 | 否 | `Σ lineDiffQty`，差异=调出数-实收数 |
| qtyInTransit | 在途总数 | integer | 系统计算 | 是 | 否 | 草稿为 0；已调出为调出总数；已调入为 0 |
| hasDifference | 是否差异 | boolean | 系统计算 | 是 | 否 | `diffQty > 0` 时为 true |
| blNo | 关联报损单号 | string | 系统生成 | 否 | 否 | 差异入库后关联 BL；无差异展示 `-` |
| expectedOutDate | 计划调出日期 | date | 用户输入/ERP 下发 | 否 | 草稿可编辑 | 不超过当前日期规则不适用，按计划日期展示 |
| expectedInDate | 计划调入日期 | date | 用户输入/ERP 下发 | 否 | 草稿可编辑 | 不早于计划调出日期 |
| remark | 备注 | string | 用户输入/ERP 下发 | 否 | 草稿可编辑 | ≤200 字符 |

## 3. 调拨明细字段（商品·调出数·实收数·差异）

| 字段编码 | 字段名称 | 类型 | 来源 | 必填 | 可编辑 | 枚举/校验/说明 |
|:--|:--|:--|:--|:--:|:--:|:--|

| productCode | 商品编码 | string | 商品档案/ERP | 是 | 草稿可编辑 | 商品快照，字母+数字 |

| spec | 规格 | string | 商品档案 | 否 | 否 | 无值展示 `-` |
| unit | 单位 | string | 商品档案 | 是 | 否 | 如 件、箱 |
| outLocationCode | 调出货位编码 | string | 用户选择/系统推荐 | 否 | 草稿可编辑 | 二期可选；未指定时由调出仓执行时确认 |
| outLocationName | 调出货位名称 | string | 货位档案 | 否 | 否 | 随货位编码带出 |
| inLocationCode | 调入货位编码 | string | 用户选择/系统推荐 | 否 | 已调出可编辑 | 调入清点或上架时确认；无值展示 `-` |
| inLocationName | 调入货位名称 | string | 货位档案 | 否 | 否 | 随货位编码带出 |
| lineTransferOutQty | 调出数量 | integer | 用户输入/ERP 下发 | 是 | 草稿可编辑 | 正整数，`>0` |
| lineActualReceiveQty | 实收数量 | integer | 调入清点 | 条件必填 | 已调出可编辑 | `0≤实收数量≤调出数量` |

| lineInTransitQty | 明细在途数量 | integer | 系统计算 | 是 | 否 | 已调出后等于调出数量；调入完成后为 0 |

| lineRemark | 行备注 | string | 用户输入 | 否 | 是 | ≤200 字符 |

## 5. 枚举值

### 5.1 调拨状态 `status`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| DRAFT | 草稿 | 已创建，尚未调出确认 |
| TRANSFERRED_OUT | 已调出（在途） | 调出仓已扣现存，库存进入调拨在途 |
| TRANSFERRED_IN | 已调入（完成） | 调入确认完成，在途已消减 |

### 5.2 调拨类型 `transferType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| WAREHOUSE_TRANSFER | 仓间调拨 | WMS 内两个仓库之间调拨 |
| ERP_TRANSFER | ERP 下发调拨 | ERP 下发调拨需求到 WMS 执行 |

### 5.3 来源 `sourceType`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| WMS_MANUAL | WMS 手动 | 仓库主管在 WMS 创建 |
| ERP_ISSUED | ERP 下发 | ERP 调拨单下发到 WMS |

### 5.4 明细状态 `lineStatus`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| PENDING_OUT | 待调出 | TR 草稿阶段，尚未调出确认 |
| IN_TRANSIT | 在途 | 已调出，等待调入清点 |
| RECEIVED | 已收货 | 实收=调出，正常完成 |
| DIFFERENCE | 差异完成 | 实收<调出，已生成 BL 口径 |

### 5.5 差异原因 `differenceReason`

| 值 | 展示 | 说明 |
|:--|:--|:--|
| TRANSFER_LOSS | 调拨损耗 | 到货差异自动生成 BL 时使用 |

## 6. 计算字段

| 字段 | 公式 | 说明 |
|:--|:--|:--|
| 调出总数 | `Σ lineTransferOutQty` | TR 调出数量合计 |
| 实收总数 | `Σ lineActualReceiveQty` | 调入清点确认的实收合计 |
| 明细差异数量 | `lineTransferOutQty - lineActualReceiveQty` | 仅调入确认时计算 |
| 差异总数 | `Σ lineDiffQty` | 大于 0 时触发 BL |
| 在途总数 | 草稿=0；已调出=`transferOutQty`；已调入=0 | 在途不计入可用，不可销售 |
| 正常完成条件 | `actualReceiveQty = transferOutQty` | 不生成 BL |
| 差异完成条件 | `0≤actualReceiveQty<transferOutQty` | 生成 BL，原因=调拨损耗 |

## 7. 校验时机

| 时机 | 校验范围 |
|:--|:--|
| 保存草稿 | 调出仓、调入仓不能相同；调出数量为正整数；ERP 来源字段只读 |
| 确认调出 | 调出仓库存可用数量足够；明细调出数量完整；状态必须为草稿 |
| 登记实收 | 实收数量为整数，且 `0≤实收≤调出数量` |
| 确认调入 | 所有明细已录入实收数量；状态必须为已调出（在途） |
| 差异生成 BL | 差异总数>0 时生成 BL，原因固定为调拨损耗，关联 TR 单号 |
