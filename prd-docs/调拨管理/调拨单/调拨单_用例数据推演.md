# 调拨单_用例数据推演

> 角色：用例数据推演 | 类型：执行作业单
> 使用 2026 年示例数据，覆盖正常调拨在途流转和到货差异生成 BL 两个核心走账场景。

## 1. 基础数据

| 类型 | 编码 | 名称 | 说明 |
|:--|:--|:--|:--|
| 仓库 | WH-SH-01 | 上海一仓 | 启用 |
| 仓库 | WH-BJ-01 | 北京一仓 | 启用 |
| 仓库 | WH-SZ-01 | 深圳仓 | 启用 |
| 仓库 | WH-CD-01 | 成都仓 | 启用 |
| 商品 | SKU004 | 得力多功能计算器 | 单位：台 |
| 商品 | SKU002 | 晨光按动式中性笔黑色 | 单位：支 |

## 2. 公式口径

| 项 | 公式/规则 |
|:--|:--|
| 可用 | `现存 - 占用 - 冻结` |
| 调出确认 | 调出仓现存-N，调拨在途+N |
| 调入确认正常 | 调入仓现存+N，调拨在途-N |
| 差异确认 | 调入仓现存+R，调拨在途-N，生成 BL 数量=`N-R` |
| 在途限制 | 在途不可销售，不允许被销售订单占用 |

> N=调出数量，R=实收数量。本套件示例不涉及冻结数量，冻结默认为 0。

## 3. 用例 1：正常调拨，在途完整流转

### 3.1 输入

| 字段 | 值 |
|:--|:--|
| 调拨单号 | TR20260706-0001 |
| 调拨类型 | 仓间调拨 |
| 来源 | WMS 手动 |
| 调出仓 | WH-SH-01 上海一仓 |
| 调入仓 | WH-BJ-01 北京一仓 |
| 商品 | SKU004 得力多功能计算器 |
| 调出数量 N | 100 |
| 实收数量 R | 100 |

### 3.2 初始库存快照

| 仓库 | SKU | 现存 | 占用 | 冻结 | 可用 | 调拨在途 |
|:--|:--|--:|--:|--:|--:|--:|
| 上海一仓 | SKU004 | 500 | 30 | 0 | 470 | 0 |
| 北京一仓 | SKU004 | 200 | 20 | 0 | 180 | 0 |

### 3.3 调出确认后

| 仓库 | SKU | 现存 | 占用 | 冻结 | 可用 | 调拨在途 | 变化说明 |
|:--|:--|--:|--:|--:|--:|--:|:--|
| 上海一仓 | SKU004 | 400 | 30 | 0 | 370 | 0 | 现存-100，可用-100 |
| 北京一仓 | SKU004 | 200 | 20 | 0 | 180 | 100 | 待收在途+100，不可销售 |

| 单据 | 状态 | 库存流水 |
|:--|:--|:--|
| TR20260706-0001 | 已调出（在途） | 生成调出 FL，记录来源 TR |

### 3.4 调入确认后

| 仓库 | SKU | 现存 | 占用 | 冻结 | 可用 | 调拨在途 | 变化说明 |
|:--|:--|--:|--:|--:|--:|--:|:--|
| 上海一仓 | SKU004 | 400 | 30 | 0 | 370 | 0 | 不再变化 |
| 北京一仓 | SKU004 | 300 | 20 | 0 | 280 | 0 | 现存+100，可用+100，在途-100 |

| 单据 | 状态 | 差异 | BL |
|:--|:--|--:|:--|
| TR20260706-0001 | 已调入（完成） | 0 | 不生成 |

### 3.5 校验点

| 校验 | 结果 |
|:--|:--|
| 调出后上海一仓现存 | `500 - 100 = 400` |
| 调出后北京一仓在途 | `0 + 100 = 100` |
| 调入后北京一仓现存 | `200 + 100 = 300` |
| 调入后北京一仓可用 | `300 - 20 - 0 = 280` |
| 调入后在途 | `100 - 100 = 0` |

## 4. 用例 2：到货差异，生成报损单 BL

### 4.1 输入

| 字段 | 值 |
|:--|:--|
| 调拨单号 | TR20260706-0002 |
| 调拨类型 | ERP 下发调拨 |
| 来源 | ERP 下发 |
| ERP 来源单号 | ERP-TR-20260706-009 |
| 调出仓 | WH-SZ-01 深圳仓 |
| 调入仓 | WH-CD-01 成都仓 |
| 商品 | SKU002 晨光按动式中性笔黑色 |
| 调出数量 N | 80 |
| 实收数量 R | 76 |

### 4.2 初始库存快照

| 仓库 | SKU | 现存 | 占用 | 冻结 | 可用 | 调拨在途 |
|:--|:--|--:|--:|--:|--:|--:|
| 深圳仓 | SKU002 | 300 | 10 | 0 | 290 | 0 |
| 成都仓 | SKU002 | 120 | 5 | 0 | 115 | 0 |

### 4.3 调出确认后

| 仓库 | SKU | 现存 | 占用 | 冻结 | 可用 | 调拨在途 | 变化说明 |
|:--|:--|--:|--:|--:|--:|--:|:--|
| 深圳仓 | SKU002 | 220 | 10 | 0 | 210 | 0 | 现存-80，可用-80 |
| 成都仓 | SKU002 | 120 | 5 | 0 | 115 | 80 | 待收在途+80，不可销售 |

| 单据 | 状态 | 库存流水 |
|:--|:--|:--|
| TR20260706-0002 | 已调出（在途） | 生成调出 FL，记录来源 TR |

### 4.4 差异入库后

| 仓库 | SKU | 现存 | 占用 | 冻结 | 可用 | 调拨在途 | 变化说明 |
|:--|:--|--:|--:|--:|--:|--:|:--|
| 深圳仓 | SKU002 | 220 | 10 | 0 | 210 | 0 | 不再变化 |
| 成都仓 | SKU002 | 196 | 5 | 0 | 191 | 0 | 现存+76，可用+76，在途-80 |

| 单据 | 状态 | 差异数量 | 关联单据 |
|:--|:--|--:|:--|
| TR20260706-0002 | 已调入（完成） | 4 | BL20260706-0001 |

### 4.5 BL 生成口径

| 字段 | 值 |
|:--|:--|
| 报损单号 | BL20260706-0001 |
| 来源单据 | TR20260706-0002 |
| 原因 | 调拨损耗 |
| SKU | SKU002 |
| 报损数量 | `80 - 76 = 4` |
| 说明 | BL 独立套件待补，本文件只定义触发和关联口径 |

### 4.6 校验点

| 校验 | 结果 |
|:--|:--|
| 调出后深圳仓现存 | `300 - 80 = 220` |
| 调出后成都仓在途 | `0 + 80 = 80` |
| 差异数量 | `80 - 76 = 4` |
| 差异入库后成都仓现存 | `120 + 76 = 196` |
| 差异入库后成都仓可用 | `196 - 5 - 0 = 191` |
| 差异入库后成都仓在途 | `80 - 80 = 0` |
| BL 数量 | `4`，原因=`调拨损耗` |

## 5. 用例 3：调出仓库存不足

| 输入 | 值 |
|:--|:--|
| 调拨单 | TR20260706-0003 |
| 调出仓可用 | 50 |
| 调出数量 | 60 |

| 判断 | 结果 |
|:--|:--|
| 可用校验 | `50 ≥ 60` 不成立 |
| 期望结果 | 阻断确认调出，TR 保持草稿 |
| 库存变化 | 现存、占用、可用、在途均不变 |

## 6. 用例 4：实收大于调出

| 输入 | 值 |
|:--|:--|
| 调拨单 | TR20260706-0004 |
| 调出数量 | 80 |
| 实收数量 | 82 |

| 判断 | 结果 |
|:--|:--|
| 数量校验 | `82 ≤ 80` 不成立 |
| 期望结果 | 阻断确认调入，提示实收数量不能大于调出数量 |
| 库存变化 | TR 保持已调出（在途），不生成 BL |
| 不确定性 | context 未定义超收处理，需后续补规则 |

## 7. 走查清单

| 检查项 | 预期 |
|:--|:--|
| TR 前缀 | 符合 `TR{YYYYMMDD}-{4位序号}` |
| BL 前缀 | 差异报损符合 `BL{YYYYMMDD}-{4位序号}` |
| 状态机 | 草稿 → 已调出（在途） → 已调入（完成） |
| 调出确认 | 调出仓现存-N，调拨在途+N |
| 在途限制 | 在途不可销售，不计入可用 |
| 正常调入 | 调入仓现存+N，调拨在途-N |
| 差异调入 | 调入仓现存+R，调拨在途-N，生成 BL |
| BL 原因 | 固定为调拨损耗 |
| BL 套件 | 本次不产出完整套件，仅记录触发口径 |
