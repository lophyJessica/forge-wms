# 02-上下游关联系统

> 强盛科技进销存体系中的 WMS 定位与数据流。

## 系统关系

```
┌──────────────┐  采购订单下发   ┌──────────────┐  入库/出库数据  ┌──────────┐
│  进销存 ERP   │ ─────────────→ │     WMS       │ ─────────────→ │ 财务系统  │
│ 采购/销售模块  │ ←───────────── │  仓储管理系统   │ ←───────────── │          │
└──────────────┘  收货/发货回执  └──────────────┘  库存凭证/应付  └──────────┘
                                       │
                         ┌──────────┐  │  ┌──────────┐
                         │ PDA 终端  │←─┘  │   TMS    │
                         │ (收货/拣货 │      │ (交运对接)│
                         │  盘点)     │      └──────────┘
                         └──────────┘
```

## 各系统对接

| 系统 | 方向 | 数据 | 触发时机 |
|:--|:--|:--|:--|
| 进销存ERP | → WMS | 采购订单 PO | 审核通过后下发 |
| 进销存ERP | → WMS | 销售订单 SO | 审核后下发，触发波次 |
| WMS | → 进销存ERP | 收货完成回执 | 入库确认 |
| WMS | → 进销存ERP | 发货完成回执 | 出库交运 |
| WMS | → 财务 | 入库凭证(应付) | 采购入库确认 |
| WMS | → 财务 | 出库凭证(应收) | 销售出库确认 |
| WMS | → TMS | 交运单 | 包装完成推送 |
| PDA | ↔ WMS | 收货/上架/拣货/复核/盘点 | 实时API |
