# 08-调拨流程

> 强盛科技二期内容。仓间调拨 + 与进销存 ERP 调拨单对接。

## 调拨类型

| 类型 | 说明 |
|:--|:--|
| 仓间调拨 | WMS 内两个仓库之间 |
| ERP 调拨 | 进销存 ERP 下发调拨单到 WMS 执行 |

## 流程（分步调拨）

```
ERP/WMS创建调拨单(TR) → 调出库确认出库(扣现存,改在途)
→ 调入库收货清点 → 实收=调出? → 是:确认入库(加现存,消在途)
                                → 否:按实收R入库(现存+R、在途-R) + 差异D=N-R转「差异待核销」(在途-D、差异待核销+D) + 生成报损单(BL,待审核)
```

## 业务规则
- 调拨在途库存不可销售（状态=在途）
- 到货差异自动生成报损单，原因标记"调拨损耗"
- 调入仓库仅按实收数量入库

## 调拨差异台账(差异待核销中间态 · 权威口径)

解决"调拨损耗由谁扣、何时扣"，避免 TR 在途清零与 BL 核销双重扣减现存。

调入确认时(实收 R,调出 N,差异 D=N−R,D≥0):
1. 调入仓 现存 += R
2. 在途 −= R(只清实收对应部分)
3. 差异 D:在途 −= D,同时 调入仓「差异待核销」+= D(挂调入仓名下的不可用中间态)
4. 系统自动生成调拨损耗报损单 BL(状态=待审核),关联该 D 件、原因固定 TRANSFER_LOSS
5. 调拨单 TR 置「已调入(完成)」——TR 完成 ≠ 差异已核销

BL 审核核销时:
6. 调入仓「差异待核销」 −= D
7. 生成库存流水 FL(调拨损耗核销类型,数量 −D,作用于「差异待核销」口径)
8. 现存不因调拨损耗二次扣减;系统总现存此刻才实现 −D

BL 审核驳回时:
9. 「差异待核销」D 件保持挂账(实物已缺、不回补现存)
10. 允许对该差异重新生成 BL 或转人工核实;差异待核销在重新核销前持续挂账

不变量:现存仅在「调出确认(调出仓−N)」「调入确认(调入仓+R)」两处因调拨而变;调拨损耗 BL 核销绝不二次扣现存。「差异待核销」不计可用、不可销售、不可出库占用。
