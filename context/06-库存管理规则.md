# 06-库存管理规则

> 强盛科技 WMS 库存核心规则：增/减/占/放/冻/解。

## 库存状态

| 状态 | 含义 | 触发 |
|:--|:--|:--|
| 可用 | 可销售/可出库 | 上架完成、解冻 |
| 占用 | 销售订单锁定 | SO审核 |
| 冻结 | 质检中/盘点中/报损 | QC/盘点/报损触发 |
| 在途 | 调拨途中 | 调出库确认→调入库确认之间 |
| 差异待核销 | 调拨到货短少、待报损核销的中间挂账(不可用/不可销售) | 调拨差异确认调入 → 报损单(调拨损耗)审核核销之间 |

## 库存动作

| 动作 | PC端 | PDA端 | 库存影响 |
|:--|:--|:--|:--|
| 入库+ | 采购收货确认 | PDA上架确认 | 可用+N |
| 出库- | 包装完成(PKG)过账 | 拣货扫描(不过账) | 现存-N、占用-N |
| 占用 | 销售订单审核 | — | 可用转占用 |
| 释放 | 订单取消 | — | 占用转可用 |
| 冻结 | 质检/盘点锁库 | — | 可用转冻结 |
| 解冻 | 质检合格/盘点完成 | — | 冻结转可用 |
| 调拨出 | 调拨单调出确认 | — | 调出仓 现存−N、在途+N |
| 调拨入 | 调拨单调入确认 | — | 调入仓 现存+R；在途−R；差异D=N−R:在途−D、差异待核销+D |
| 差异核销 | 报损单(调拨损耗)审核核销 | — | 差异待核销−D；现存不变 |

## 公式
- 可用 = 现存 - 占用 - 冻结
- 现存 = 期初 + Σ入库 - Σ出库 + Σ盘盈 - Σ盘亏
- 安全库存 / 最低库存来源 = 商品档案 `safetyStock` / `minimumStock`（库存预警阈值的唯一 SSOT，预警页只读引用、不自建阈值）
- 在途、差异待核销 均为独立不可用口径,不进「可用」公式、不可销售、不可被出库占用。
- 调拨守恒:调出前系统总现存 = 调出后(各仓现存合计 + 在途 + 差异待核销);直到调拨损耗 BL 审核核销,系统总现存才实现 −D。

## 库存流水 FL
每次库存变动(含调拨出/调拨入/差异核销)生成一条：时间/仓库/货位/商品/变动类型/变动数量(±)/变动后现存/变动后在途/变动后差异待核销。调拨损耗核销用独立变动类型,不复用盘点盘亏。
