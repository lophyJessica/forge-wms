# 强盛科技 WMS 项目背景

> 基于真实 B 端 WMS 产品经验，以强盛科技为演示品牌还原。

## 1. 业务痛点

强盛科技从华强北单档口发展到前店后仓+区域代理分销模式，仓储管理面临：

- **手工台账对不上**：仓管员用 Excel，年底盘点差异率 8-15%
- **作业效率低**：一张采购入库从收货到上架平均 45 分钟
- **库存不透明**：不知道哪些可用、哪些被占用、哪些在质检中
- **多仓失控**：6 个仓库调拨靠微信群，货调出去系统不自知
- **与进销存 ERP 割裂**：ERP 有采购订单，仓库收货后需人工补录

## 2. 项目目标

| 维度 | 目标 |
|:--|:--|
| 库存准确率 | 从 85-92% → ≥ 99% |
| 入库效率 | 45分钟/单 → 15分钟/单（PDA 扫码） |
| 库存可视化 | 实时区分可用/占用/冻结/在途 |
| 多仓统一 | 6仓统一管理，跨仓调拨 |
| ERP 对接 | 采购订单自动下发，收货结果自动回传 |

## 3. 核心角色

| 角色 | 职责 |
|:--|:--|
| 仓管员 | PDA 收货/上架/拣货/复核/发货 |
| 仓库主管 | 审核盘点差异、审批调拨、查看报表 |
| 采购专员(进销存ERP) | 下发采购订单，查看收货进度 |
| 财务人员 | 基于 WMS 数据触发付款和对账 |

## 4. 本期范围（MVP）

**包含：** 基础数据、入库(收货/质检/上架)、出库(波次/拣货/播种/复核/包装/交运)、库存管理、盘点、PDA端

**本期不做：** 调拨(二期)、波次策略引擎(三期)、物流公司对接、多货主
